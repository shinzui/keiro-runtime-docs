---
title: pgmq
description: A PostgreSQL-native message queue — the queue substrate of the keiro runtime, via pgmq-hs.
---

pgmq is a PostgreSQL-native message queue, and **pgmq-hs** is the first-party
Haskell client for it. In the keiro runtime family, pgmq is the queue substrate:
applications create queues, send JSON messages, lease visible messages for a
visibility timeout, then archive or delete the messages after work completes.
shibuya workers consume pgmq through the
[shibuya pgmq adapter](/docs/integrations/shibuya-pgmq-adapter), while keiro's
job layer builds typed background jobs on top of the same primitives.

pgmq-hs is split by concern. `pgmq-core` owns validated names and message
types. `pgmq-hasql` is the direct Hasql client. `pgmq-effectful` lifts the same
operations into an `effectful` dynamic effect and can emit OpenTelemetry spans.
`pgmq-config` reconciles declared queue topology at startup. `pgmq-migration`
exports the native component that installs the pgmq 1.11.0 schema without relying on the PostgreSQL
extension and can explicitly adopt verified predecessor history.

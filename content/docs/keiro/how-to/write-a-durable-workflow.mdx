description: "Compose step, sleepNamed, awakeableNamed, and child workflows into a durable function, run it with runWorkflow, set run options, and avoid the common pitfalls."

You want to write a long-running process as one function whose checkpoints survive a crash, a
redeploy, or an idle wait. This guide assembles a durable workflow from the four primitives and runs
it. For the concept, see [Understanding durable execution](/docs/keiro/explanation/durable-execution);
for every signature, the [durable workflows reference](/docs/keiro/reference/durable-workflows).

## Import the surface

The umbrella `Keiro` module does not re-export workflows — import the submodules directly:

```haskell
import Keiro.Workflow           -- step, runWorkflow/runWorkflowWith, WorkflowOutcome, the journal codec
import Keiro.Workflow.Sleep     -- sleepNamed / sleep
import Keiro.Workflow.Awakeable -- awakeableNamed, signalAwakeable, cancelAwakeable
import Keiro.Workflow.Child     -- spawnChild, awaitChild, cancelChild
```

## Compose the primitives

A workflow is an `Eff (Workflow : es) a`. Use `step` for any side effect, and the three waits in line:

```haskell
orderFulfillmentWorkflow :: (Workflow :> es, Store :> es, IOE :> es) => OrderId -> Eff es Text
orderFulfillmentWorkflow orderId = do
  _reservation <- step (StepName "reserve-inventory") (reserveInventory orderId)
  sleepNamed (StepName "cooling-off") 300                       -- a durable 5-minute wait
  (_awkId, awaitPayment) <- awakeableNamed (StepName "payment-webhook")
  payment <- awaitPayment                                       -- wait for an external signal
  _charge <- step (StepName "charge") (chargeCard orderId payment)
  childHandle <- spawnChild shipOrderWorkflowName (shipChildId orderId) (shipOrderWorkflow orderId)
  awaitChild childHandle                                        -- wait for a child workflow
```

- **`step name action`** runs `action` once and journals its JSON result; a replay returns the
  recorded value without re-running. Anything `ToJSON`/`FromJSON` works as a result.
- **`sleepNamed name delta`** durably pauses (a `keiro_timers` row) and resumes when a timer worker
  fires it. No thread is held.
- **`awakeableNamed name`** returns a deterministic id and an `await`; the workflow suspends until an
  external caller runs `signalAwakeable`.
- **`spawnChild` / `awaitChild`** spawn and wait on a child workflow with its own journal.

<Callout type="warn">
**Name your steps, and keep names stable.** A step's identity is its `StepName`, not its position. The
named forms (`step`, `sleepNamed`, `awakeableNamed`) survive code edits that insert or reorder
unrelated code. The ordinal convenience forms (`sleep`, `awakeable`) derive names from call order and
are only conditionally safe — avoid them for anything that must survive a code change mid-flight.
</Callout>

## Run it

`runWorkflow` runs (or resumes) the workflow and returns a [`WorkflowOutcome`](/docs/keiro/reference/durable-workflows#running-a-workflow):

```haskell
outcome <- runWorkflow (WorkflowName "order-fulfillment") (workflowIdFor orderId) (orderFulfillmentWorkflow orderId)
case outcome of
  Completed tracking -> ...   -- finished; `tracking` is the function's result
  Suspended          -> ...   -- parked at a wait; a resume pass will continue it later
  Cancelled          -> ...   -- a parent cancelled this (child) workflow
```

Running the same `(name, id)` again replays the journal and short-circuits completed steps, so calling
`runWorkflow` is itself the resume mechanism. In production you do not call it by hand on every wait —
you [run the resume worker](/docs/keiro/how-to/run-the-workflow-resume-worker) on a loop.

## Resolve and repair the waits

- **Sleeps** fire when a timer worker drains `keiro_timers`. Run `runWorkflowTimerWorker` (it handles
  both workflow sleeps and ordinary process-manager timers in one pass) on your timer polling loop.
- **Awakeables** resolve when an external system calls `signalAwakeable awkId payload`. Compute the id
  with `deterministicAwakeableId name id label` — the signaller never needs to read the journal back.
  Repair one that will never arrive with `cancelAwakeable awkId`; the workflow's `await` then throws
  `WorkflowAwakeableCancelled`, which you can `catch` for compensation.
- **Children** must be **registered in the resume worker's registry** by name — `spawnChild` does not
  run the child inline; the worker drives it. Repair a parent stuck on a never-finishing child by
  driving or cancelling the child (`cancelChild`).

<Callout type="warn">
**A parent and its child must use distinct `WorkflowId`s.** Discovery groups by `workflow_id`, so a
shared id lets a completed child mask an unfinished parent. Derive the child id, e.g.
`WorkflowId (parentId <> "-ship")`.
</Callout>

## Set run options for long journals

Use `runWorkflowWith` to enable snapshots (so a resume does not replay every step) or telemetry. Set
fields with the **generic-lens label** — a bare record update on `snapshotPolicy` is ambiguous:

```haskell
let opts = defaultWorkflowRunOptions
             & #snapshotPolicy .~ Every 2     -- snapshot the step-result map every 2 appends
             & #metrics        .~ Just metrics
             & #tracer         .~ Just tracer
outcome <- runWorkflowWith opts name wid (orderFulfillmentWorkflow orderId)
```

The resume worker threads the same options through `WorkflowResumeOptions.runOptions`, so a resumed run
honours them too. Snapshots are advisory: a stale or corrupt one degrades to a full replay, never a
wrong result.

## Make step actions safe to re-run

Replay short-circuits a *journaled* step — but a step that **crashes before it journals** re-runs on
the next pass. So keep the action inside a `step` idempotent with respect to the outside world: pass
deterministic external ids, dedupe at the destination, or model the effect through keiro's
[outbox](/docs/keiro/explanation/the-outbox-pattern) / [idempotent
commands](/docs/keiro/how-to/make-commands-idempotent). The journal guarantees *at most once per
completed step*; your action should be safe under *at least once until it completes*.

---
title: "Rebuild a read model"
description: "Bump the schema, mark the model rebuilding, repopulate it manually, and promote it back to live."
---

You changed a read model's shape or logic and need to rebuild its table. keiro gives you the status
transitions; the repopulation itself is yours to drive.

<Callout type="info">
  Assumes you have a registered read model (see [Read model reference](/docs/keiro/reference/read-model)).
</Callout>

## Goal

Take a read model through a schema change: bump its identity, rebuild, repopulate, and promote.

## Steps

1. **Bump `version` and/or `shapeHash`** in your `ReadModel` definition so it no longer matches the
   registered row. The next `runQuery` would otherwise hard-fail with `ReadModelStaleSchema`.

2. **Mark it rebuilding.** `rebuild model` transitions the registry row to `Rebuilding`:

   ```haskell
   _ <- rebuild myReadModel       -- markRebuilding: status -> Rebuilding
   ```

3. **Repopulate the table — manually.** Replay the relevant events into the (new-shape) table
   yourself. There is no built-in replay.

4. **Promote it back to live** (or abandon):

   ```haskell
   _ <- promote myReadModel        -- markLive: status -> Live
   -- or, on failure:
   _ <- abandonRebuild myReadModel -- markAbandoned: status -> Abandoned
   ```

<Callout type="warn">
`rebuild`, `promote`, and `abandonRebuild` are status transitions **only** — there is no shadow-table
swap and no automated replay. Repopulation between `rebuild` and `promote` is your job, and while the
model is not `Live`, `runQuery` returns `ReadModelNotLive`. The research design's multi-step rebuild
protocol is **not** implemented.
</Callout>

## Verify it worked

After `rebuild`, confirm `runQuery` returns `ReadModelNotLive`. After repopulating and `promote`,
confirm `runQuery` succeeds and returns the new-shape data.

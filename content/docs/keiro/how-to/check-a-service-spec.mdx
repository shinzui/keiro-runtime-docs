---
title: "Check a service spec"
description: "Run the current keiro-dsl parser and semantic gate, interpret located errors and warnings, and fix topology, policy, identity, or persistence defects before scaffolding."
---

Run `check` before generating Haskell. It parses the whole `.keiro` file, validates every top-level
node and cross-node reference, prints located diagnostics, and exits nonzero when any diagnostic is
an error.

## Run and normalize

```bash
keiro-dsl check service.keiro
# OK

keiro-dsl check service.keiro --emit
# canonical accepted spec
```

Warnings print but exit 0. Errors use this form and exit nonzero:

```text
service.keiro:<line>: error[Code]: <message>
```

`--emit` combines parse, validation, and canonical pretty-printing. Scaffold runs the same gate and
writes nothing for an invalid spec.

## Fix the owning family

| Family | Representative diagnostics | Corrective move |
|---|---|---|
| names and syntax | `IdentHaskellKeyword`, `IdentNotConstructorSafe`, `VertexCtorCollision`, duplicate-name codes | Rename to an ASCII, Haskell-safe, collision-free identifier; remove shadowed singleton/row declarations |
| aggregate/evolution | `RegisterInitialOutOfScope`, `WriteTargetNotRegister`, `StatusMapNotTotal`, `EvtVersionMissingUpcaster`, `DuplicateUpcasterSource`, `UpcasterChainGap`, `ClockSampled` | Align declared types/references, complete or explicitly partialize the map, add contiguous upcasters with one owner per source version, inject time |
| event retirement / replay-only | `EventRetirementInProgress`, `DeprecatedEventStillEmitted`, `ReplayOnlyEmitsNothing` | Keep a live emitter while `retiring`; take the event off the *live* write path before marking it `deprecated`; give every `replay-only` transition an `emit` |
| read model | `RmShapeHashDrift`, `RmStrongInlineOnly`, `RmScopeWithoutStrong`, `RmInlineFeedUnreferenced` | Capture the printed shape, bump version for a real shape change, align consistency/feed/scope/projection ownership |
| process/router | `SagaCategoryIllegal`, `ProcessFieldBindingUnresolved`, `RouterBindingUnscoped`, `PolicyContradiction`, `AmbiguousMarkedBenign` | Use a valid category, resolve typed bindings, align node/dispatch policy, route ambiguity to retry/dead-letter evidence |
| inbox/outbox | `DispositionIncomplete`, disposition inversion codes, `TopicAffinityMismatch`, `EmitSkipMissing` | Complete unique outcomes, preserve retry/poison semantics, align contract topic, add `_ => skip` |
| workqueue/dispatch | queue divergence/group/provision codes, `WqDispositionIncomplete`, dispatch reference codes | Copy the derived queue trio, align FIFO/group/provision, complete policy, resolve read-model columns and queue fields |
| workflow/operation | duplicate label/patch codes, `WorkflowContinueAsNewNotTerminal`, `AwaitSignalMismatch`, `OperationUnresolvedRef` | Stabilize labels/patch IDs, put continuation last/top-level, align signal value and target references |

See [domain/read-side notation](/docs/keiro/reference/keiro-dsl-domain-nodes) and
[runtime/integration notation](/docs/keiro/reference/keiro-dsl-runtime-nodes) for exact closed values
and defaults.

## Treat warnings as review items

Warnings do not block the CLI, but they describe an acknowledged risk or forward policy:

- `ProcessBenignInversion`: duplicate/rejected delivery is being treated as success under a
  target-scoped runtime check;
- `PolicyDeadLetterUnused` / `AmbiguousFollowsRejectedPolicy`: node-level rejection behavior needs
  explicit operator understanding;
- `WqUnloggedDurability`: a database crash can empty the queue;
- `RmProjectionWithoutNode`: legacy projection wiring lacks registered read-model identity/helpers;
- `DeprecatedEventReplayHazard`: a deprecated event has **no** replay-only emitter, so stored history
  containing it is no longer replayable — restore an equivalent `replay-only` transition, or
  terminalize/truncate every affected stream before deployment
  ([retire an event type](/docs/keiro/how-to/retire-an-event-type));
- `EventRetirementInProgress`: the safe retained shape is in place; keep the eventual cleanup visible;
  and
- `ReplayOnlyCommandStillLive`: a `replay-only` transition has no live sibling for its
  `(source, command)` pair — the command is fully retired at that state, and the fuller procedure is
  event retirement.

Do not suppress these by changing generated code. Decide whether the spec policy is intentional.

<Callout type="warn">
`check` sees one spec. It cannot know whether live streams still contain a deprecated event or still
exercise a retired guard region — that is operational history, answered by the
[replay audit](/docs/keiro/how-to/audit-real-logs-before-a-deploy). That is why those two are warnings
rather than errors.
</Callout>

## Gate CI

```bash
keiro-dsl check service.keiro
```

The exit code is sufficient for CI. Use `/dev/stdin` for pipelines, for example:

```bash
keiro-dsl new router | keiro-dsl check /dev/stdin
```

Continue with [Scaffold and fill holes](/docs/keiro/how-to/scaffold-and-fill-holes) only after the
spec is green.

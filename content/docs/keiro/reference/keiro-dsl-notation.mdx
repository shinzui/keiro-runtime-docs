The `.keiro` notation is the typed-specification language for a keiro service. A file is
`context <name>` followed by top-level declarations and nodes; `#` begins a line comment and
whitespace is insignificant (structure comes from keywords). Every node family below is parsed,
validated, and round-tripped by the toolchain. For the *why* ŌĆö the firewall, the dangerous-decision
checks, and the authoring loop ŌĆö see [The keiro-dsl service
toolchain](/docs/keiro/explanation/the-keiro-dsl-toolchain).

<Callout type="info">
Throughout, a `HOLE` (or a hand-filled body) marks a decision the scaffolder leaves to you in a
create-if-absent module; everything else is emitted into overwrite-on-scaffold `-- @generated`
modules. No generated line ever contains a keiki symbolic operator (the firewall).
</Callout>

## Shared declarations

```keiro-dsl
context hospital-capacity

id   TransferReservationId  prefix=rsv          # an id newtype over Text
enum DivertStatus { Open=open TotalDivert=total-divert }   # Ctor=wire-spelling
rule lifeCriticalOverride : PatientAcuity -> Bool
  ex RedTag => true ; YellowTag => false ; GreenTag => false
```

- `id` declares a newtype over `Text` with a typeid-style `prefix`.
- `enum` lists constructors with their wire spelling (`Ctor=wire-spelling`).
- `rule` names a typed predicate/function with example rows (`ex ŌĆ”`) used by the harness.

## `aggregate` (+ evolution)

```keiro-dsl
aggregate Reservation
  regs
    reservationState ReservationVertex = Unrequested      # name Type = initial
  states Unrequested Held Confirmed Expired!              # trailing ! = terminal (no outgoing)

  command RequestTransferReservation { reservationId hospitalId commandId divertStatus lifeCriticalOverride:Bool }
  event   TransferReservationCreated = fields(RequestTransferReservation)
  event   TransferReservationConfirmed v2 { reservationId hospitalId triageNote:Text }
    upcast from v1 = HOLE                                 # vN>1 needs a contiguous upcaster hole
  deprecated event LegacyOpened { reservationId }         # removed from write path, still decodable

  Unrequested -- RequestTransferReservation -->
    guard divertStatus != TotalDivert || lifeCriticalOverride   # a typed Expr, scope-checked
    write reservationState := Held
    emit  TransferReservationCreated
    goto  Held

  wire kind=ctorName fields=camelCase schemaVersion=1
  projection transfer_decisions consistency=Strong key=reservationId
    status-map { Created=>held Confirmed=>confirmed }    # event-suffix => status; must be total
```

A transition is `<from> -- <Command> -->` followed by an optional `guard`, register `write`s, the
`emit`s, and `goto`. The `guard` is a typed `Expr` over the command's fields, declared enums, and
`rule`s; it is scope-checked. **Holes you fill:** the transducer body (guards/writes/emits via keiki
operators), the projection `apply` SQL, any `upcast from v(N-1)` body.

**Checked:** status-map totality (`StatusMapNotTotal`), every command on a transition is declared
(`UndeclaredCommand`), guards sample no clock (`ClockSampled`), and a versioned event has a contiguous
upcaster (`EvtVersionMissingUpcaster`).

## `process` + `timer`

```keiro-dsl
process HospitalSurge
  name "hospital-surge"
  input SurgeInput { hospitalId availableIcuBeds:Int observedAt:Time }
  correlate input.hospitalId via idText
  saga Surge stream="hospital-surge-" <> correlationId
  target Hospital
  projections [ hospitalReadiness ]
  on SurgeInput
    advance NoteSurgeThreshold { hospitalId timerId=timer.id }
    dispatch Hospital@input.hospitalId ActivateSurge { hospitalId }
      on-appended AckOk ; on-duplicate AckOk ; on-failed Retry
    schedule surgeFollowUp
  dispatch-id strategy=uuidv5 from=(name, correlationId, sourceEventId, emitIndex)   # runtime-owned; fixed
  timer surgeFollowUp
    id uuidv5 "hospital-surge-timer:" <> correlationId
    fireAt input.observedAt + 5m                          # TIME INJECTED, not sampled
    payload { kind="hospital-surge-follow-up" hospitalId }
    fire dispatch Surge@correlationId MarkSurgeTimerFired { hospitalId timerId }
      fired-event-id uuidv5 "hospital-surge-fired:" <> correlationId
      on-ok Fired ; on-reject Fired ; on-error Retry ; not-mine Retry   # on-reject Fired = benign inversion
    decode unknown-status => Cancelled
    max-attempts 5 dead-letter "surge timer exceeded ceiling"   # forces the dangerous default OFF
```

**Checked:** `fireAt` must reference a `:Time` input field (`ProcessFireAtNotInjected`); no
user-supplied dispatch-id (`ProcessDispatchIdSupplied`); the saga/target/fire targets resolve; and
`max-attempts` is explicit (a missing ceiling is the dangerous default). The benign inversions
(`on-reject Fired`, a rejected replay being benign success) pass with an informational warning. **Holes
you fill:** the `handle` body, the deadline window, the fire command, the SQL.

## `contract` / `intake` / `emit` / `publisher`

```keiro-dsl
contract emergency {
  schemaVersion 1
  discriminator messageType
  topic incidentEvents "emergency.incident.events"
  event IncidentTransferNeedDeclared on incidentEvents { incidentId: typeid "inc"; region: text; redCount: int }
}

intake incidentInbox {
  contract emergency
  topic incidentEvents
  accept IncidentTransferNeedDeclared
  bind messageId from header "keiro-message-id" required cross-check body
  bind key from kafka-key
  dedupe key messageId policy PreferIntegrationMessageId
  decode { envelope strict-required lenient-optional ; body strict schemaVersion == 1 }
  disposition {                                            # MANDATORY + COMPLETE (7 outcomes)
    processed => ackOk
    duplicate => ackOk                                     # inversion 1: replay is SUCCESS
    inProgress => retry 5s
    previouslyFailed => deadLetter "prior failure"          # inversion 2: NOT retry
    decodeFailed => deadLetter                             # inversion 3: poison, NOT unbounded retry
    dedupeFailed => deadLetter
    storeFailed => retry 5s
  }
}

emit reservationResponse {
  contract emergency ; topic hospitalEvents ; source "hospital-capacity" ; key incidentId
  map status { "confirmed" => TransferReservationAccepted  _ => skip }   # `_ => skip` is MANDATORY
  messageId derive "msg" hole ; idempotencyKey derive hole
}

publisher hospitalPublisher {
  emit reservationResponse ; ordering PerKeyHeadOfLine ; maxAttempts 10
  backoff constant 2s ; outboxId stable from messageId
}
```

- A `contract` is the cross-service message schema (topics, events, typed fields), defined once and
  referenced by both producer and consumer.
- An `intake` is the [inbox](/docs/keiro/reference/inbox): bind envelope rows, dedupe, decode, and a
  **complete** disposition table.
- An `emit` is the [outbox](/docs/keiro/reference/outbox) map; `_ => skip` is mandatory (skip-totality).
- A `publisher` is the outbox publish policy (ordering, retries, backoff, stable outbox id).

**Checked:** the disposition table is complete and the three inversions (duplicateŌćÆackOk,
previouslyFailedŌćÆdeadLetter, decodeFailedŌćÆdeadLetter) are flagged if stated the unsafe way; `_ => skip`
is present; contract/topic/event coupling resolves; `publisher ŌåÆ emit` resolves.

## `workqueue` / `dispatch`

```keiro-dsl
workqueue reservation_work {
  queue logical = "hospital_capacity.reservation_work"
  derive physical = "hospital_capacity_reservation_work"   # captured fixture; validator re-derives + checks drift
         dlq = "hospital_capacity_reservation_work_dlq"
         table = "pgmq.q_hospital_capacity_reservation_work"
  payload ReservationWorkItem { reservationId -> "reservation_id" text required }
  retry maxRetries = 3 delay = 5s dlq = on                 # dlq=on needs maxRetries>=1
  disposition {
    storeFailure -> retry 5s                               # transient: MUST retry
    decodeFailure -> deadLetter                            # poison: MUST dead-letter
    commandRejected -> deadLetter
    onCodecReject -> deadLetter
  }
}

dispatch reservation_work_dispatch {
  source readModel = accepted_transfer_needs key = reservationId
  fanout body = resolveTransferCandidates                  # effectful 1->N ŌĆö HOLE
  dedup key = reservationId
        seenIn readModel = transfer_decisions field = reservation_id
        seenIn queue = reservation_work field = reservation_id   # raw-SQL ŌĆö HOLE
  enqueue to = reservation_work
}
```

This is the [background-jobs (keiro-pgmq)](/docs/keiro/reference/pgmq-jobs) surface. The `derive`
block captures the physical queue/table names; the validator re-derives them from the logical name and
flags any drift (`WqPhysicalDivergence`). **Checked:** physical-name divergence, the store/decode
inversions, `dlq = on` requires `maxRetries >= 1`, and the `enqueue to` resolves. **Holes:** the
`fanout` body and the dedup raw-SQL.

## `workflow` / `operation`

```keiro-dsl
workflow HospitalTransferReservation
  name "hospital-transfer-reservation"
  in ReservationWorkflowInput { reservationId:Id hospitalId:Id }
  out ReservationWorkflowSummary
  id from input.reservationId via idText
  body                                                     # ORDERED; replay matches on label
    step  create-transfer-hold      -> ReservationHold
    await reservation-confirmation  -> ReservationConfirmation
    sleep cooling-off after coolingOffDelay               # TIME INJECTED
    child ship-order id input via shipChildId -> Text     # child id MUST differ from parent's

operation SignalReservationConfirmation
  signal reservation-confirmation of HospitalTransferReservation   # MUST match an await label
    key from reservationId via reservationWorkflowId
    value ReservationConfirmation
operation RunReservationWorkflow
  run HospitalTransferReservation
    input ReservationWorkflowInput
    outcome -> ReservationWorkflowRun
```

A workflow `body` is **ordered**: on replay each entry matches on its label (so the
[journal](/docs/keiro/reference/durable-workflows) is stable). The four step shapes are `step`,
`await`, `sleep` (time injected), and `child` (whose id must differ from the parent's). The four
operation shapes are `command on <Agg> ŌĆ”`, `query <ReadModel> ŌĆ”`, `signal <label> of <Wf> ŌĆ”`, and
`run <Wf> ŌĆ”`. **Checked:** every `signal <label> of <wf>` matches an `await <label>` of that workflow
(`AwaitSignalMismatch`) ŌĆö otherwise the awakeable id never matches and the workflow waits forever.

## The CLI

Run from the keiro repo root. The tool is `keiro-dsl` (`cabal run keiro-dsl --`).

```bash
keiro-dsl parse    <file.keiro>                # parse + pretty-print it back (proves it's a real spec)
keiro-dsl check    <file.keiro>                # validate; print diagnostics; exit non-zero on any error
keiro-dsl scaffold <file.keiro> --out DIR      # emit @generated modules + create-if-absent holes
keiro-dsl diff     --since <git-ref> <file.keiro>   # classify changes ADDITIVE/BREAKING since a ref
```

<TypeTable
  type={{
    parse: { type: "FILE", description: "Parses the spec and prints it pretty-printed back. parse . pretty == id is a property. A parse error is line-numbered; exits non-zero." },
    check: { type: "FILE", description: "Runs validateSpec. Prints each Diagnostic (line-numbered). Prints OK and exits 0 only if no diagnostic has Error severity; warnings pass." },
    scaffold: { type: "FILE --out DIR", description: "Emits the modules. Generated modules are overwritten unconditionally; HoleStub modules (Holes.hs / ProcessHoles.hs) are written only if absent." },
    diff: { type: "--since GIT-REF FILE", description: "Parses the spec at <ref> (via git show) and on disk, diffs them, prints each change as ADDITIVE/BREAKING, and exits non-zero if any change is breaking." },
  }}
/>

<Callout type="info">
Exit codes are the gate: `check` and `diff` are meant to run in CI. A non-zero `check` names the rule
and line ŌĆö fix the **spec**, not the generated code. A non-zero `diff` means an on-disk event payload
could now fail to decode ŌĆö add a versioned event + `upcast from v(N-1) = HOLE`, or a `deprecated
event`, until it reports `ADDITIVE`. Use `/dev/stdin` as `FILE` to read a spec from stdin.
</Callout>

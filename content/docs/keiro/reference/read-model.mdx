## `ConsistencyMode` and `PositionWaitOptions`

```haskell
data ConsistencyMode
  = Strong
  | Eventual
  | PositionWait !PositionWaitOptions
  deriving stock (Generic, Eq, Show)
```

<Callout type="warn">
In `runQuery`, **`Strong` and `Eventual` are behaviorally identical** — both skip waiting. Only
`PositionWait` with a `Just target` blocks (polling `subscriptions.last_seen`). For read-your-writes,
pass the `GlobalPosition` a command just returned as the `PositionWait` target.
</Callout>

## `ReadModelError`

```haskell
data ReadModelError
  = ReadModelStaleSchema !Text !Int !Int !Text !Text            -- name, expVer, foundVer, expHash, foundHash
  | ReadModelWaitTimeout !Text !GlobalPosition !GlobalPosition  -- name, target, last observed
  | ReadModelNotLive     !Text !ReadModelStatus                 -- name, current status
  deriving stock (Generic, Eq, Show)
```

## Query functions

```haskell
runQuery     :: (IOE :> es, Store :> es) => Maybe KeiroMetrics -> ReadModel q r -> q -> Eff es (Either ReadModelError r)
runQueryWith :: (IOE :> es, Store :> es) => Maybe KeiroMetrics -> ConsistencyMode -> ReadModel q r -> q -> Eff es (Either ReadModelError r)
waitFor      :: (IOE :> es, Store :> es) => Maybe KeiroMetrics -> PositionWaitOptions -> ReadModel q r -> GlobalPosition -> Eff es (Either ReadModelError ())
```

`runQueryWith` does, in order: `ensureReadModel` (which calls `registerReadModel` then
`validateMetadata`); if the schema and liveness are OK, `waitIfNeeded` for the mode; then run `query`
in a transaction. On schema drift it hard-fails with `ReadModelStaleSchema`; while a model is not
`Live` it fails with `ReadModelNotLive`.

Each of these takes the opt-in [metrics](/docs/keiro/reference/telemetry#metrics-surface) handle as
its leading argument. When a `Strong`/`AwaitPosition` wait hits its deadline, `waitFor` records the
[`keiro.projection.wait.timeouts`](/docs/keiro/reference/telemetry#metric-catalogue) counter; pass
`Nothing` to disable it. (These signatures gained the `Maybe KeiroMetrics` argument with the metrics
surface; 0.1.0.0 callers pass `Nothing`.)

## `readSubscriptionPosition`

Read a subscription's checkpoint — the global position its projection has caught up to — or `Nothing`
if it has none yet. Used internally by `waitFor` and by
[`recordProjectionLag`](/docs/keiro/reference/projection#recordprojectionlag).

```haskell
readSubscriptionPosition :: (Store :> es) => Text -> Eff es (Maybe GlobalPosition)
```

## The registry

```haskell
data ReadModelStatus = Live | Rebuilding | Paused | Abandoned
  deriving stock (Generic, Eq, Show)

data ReadModelMetadata = ReadModelMetadata
  { name        :: !Text
  , version     :: !Int
  , shapeHash   :: !Text
  , lastBuiltAt :: !(Maybe UTCTime)
  , status      :: !ReadModelStatus
  }

registerReadModel :: (Store :> es) => Text -> Int -> Text -> Eff es ReadModelMetadata
lookupReadModel   :: (Store :> es) => Text -> Eff es (Maybe ReadModelMetadata)
markRebuilding    :: (Store :> es) => Text -> Int -> Text -> Eff es ReadModelMetadata
markLive          :: (Store :> es) => Text -> Int -> Text -> Eff es ReadModelMetadata
markAbandoned     :: (Store :> es) => Text -> Int -> Text -> Eff es ReadModelMetadata
```

`registerReadModel` is an idempotent `INSERT … ON CONFLICT (name) DO NOTHING` that returns the
existing row **unchanged** — it never overwrites the stored `version`/`shapeHash`. That is exactly
what lets `validateMetadata` detect drift. An unrecognized stored status decodes defensively to
`Paused`.

## Rebuild lifecycle — skeleton only

```haskell
rebuild        :: (Store :> es) => ReadModel q r -> Eff es ReadModelMetadata  -- -> markRebuilding
promote        :: (Store :> es) => ReadModel q r -> Eff es ReadModelMetadata  -- -> markLive
abandonRebuild :: (Store :> es) => ReadModel q r -> Eff es ReadModelMetadata  -- -> markAbandoned
```

<Callout type="warn">
These are thin wrappers over status transitions **only**. There is **no shadow-table swap and no
automated replay** — repopulating the table between `rebuild` and `promote` is manual, and `runQuery`
returns `ReadModelNotLive` while a model is not `Live`. See
[Rebuild a read model](/docs/keiro/how-to/rebuild-a-read-model).
</Callout>

## The `keiro_read_models` table

```sql
CREATE TABLE IF NOT EXISTS keiro_read_models (
  name          TEXT PRIMARY KEY,
  version       BIGINT NOT NULL,
  shape_hash    TEXT NOT NULL,
  last_built_at TIMESTAMPTZ,
  status        TEXT NOT NULL,           -- no CHECK constraint on status
  updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

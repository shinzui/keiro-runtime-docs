This page owns `aggregate`, its nested `projection` and `snapshot`, and top-level `readmodel`.
Shared `id`, `enum`, and `rule` declarations are summarized in the [notation
overview](/docs/keiro/reference/keiro-dsl-notation).

## Aggregate

```keiro-dsl
aggregate Reservation
  regs
    reservationId TransferReservationId = placeholder
    reservationState ReservationVertex = Unrequested
    note Text = "not requested"
    attempts Int = 0
  states Unrequested Held Confirmed Expired!

  command RequestTransferReservation { reservationId hospitalId divertStatus lifeCriticalOverride:Bool }
  event TransferReservationCreated = fields(RequestTransferReservation)
  event TransferReservationConfirmed v2 { reservationId hospitalId triageNote:Text }
    upcast from v1 = HOLE
  retiring event SoonToGo { reservationId }
  deprecated event LegacyOpened { reservationId }

  Unrequested -- RequestTransferReservation -->
    guard divertStatus != TotalDivert || lifeCriticalOverride
    write reservationState := Held
    emit TransferReservationCreated
    goto Held

  replay-only Unrequested -- OpenLegacy -->
    write reservationState := Held
    emit LegacyOpened
    goto Held

  wire kind=ctorName fields=camelCase schemaVersion=1
```

### Registers, states, and expressions

The first state is initial; `!` marks a terminal state, which may not have outgoing transitions.
Scaffoldable register/explicit field types are `Text`, `Int`, `Bool`, the generated
`<Aggregate>Vertex`, and declared `id`/`enum` types. Initial forms are:

- quoted string for `Text`;
- signed in-range integer for `Int`;
- `True`/`False` for `Bool`;
- an in-domain enum/state constructor; or
- bare `placeholder` for an ID register.

Expression precedence is `||`, then `&&`, then `==`, `!=`, `<`, `<=`, `>`, `>=`. Atoms are booleans
or names resolved as command fields, registers, enum constructors, or rules. Time may not be sampled.

A transition has one source/command, optional guard, zero or more writes/emits, and exactly one
`goto`. Durable state changes must emit history; the generated harness applies Keiki replay-safety
validation to the hand-filled transducer.

### Transition modes: `live` and `replay-only`

A transition is **live** by default. A `replay-only` prefix on the transition line marks it as
serving inversion only:

```keiro-dsl
replay-only Held -- ConfirmReservation --> write reservationState := Confirmed
                                         ; emit TransferReservationConfirmed
                                         ; goto Confirmed
```

It lowers to `B.replayOnly` in the scaffolded transducer skeleton â€” keiki's `ReplayOnly` `EdgeMode`.
The edge is never taken by a new command; it exists so events emitted under a retired rule keep an
inverting edge, and its `update` defines how that history folds today. The pretty-printer round-trips
the prefix, so a `diff`-computed twin can be pasted back into the spec verbatim.

Two discipline rules:

| Code | Severity | Rule |
|---|---|---|
| `ReplayOnlyEmitsNothing` | error | a replay-only transition with no `emit` can invert nothing |
| `ReplayOnlyCommandStillLive` | warning | no live sibling for the `(source, command)` pair â€” the command is fully retired there, and the fuller procedure is event retirement |

The transition **mode** is part of the fold fingerprint, because flipping it changes replay
attribution. Use it via
[Tighten a guard with a replay-only twin](/docs/keiro/how-to/tighten-a-guard-with-a-replay-only-twin)
and [Retire an event type](/docs/keiro/how-to/retire-an-event-type).

### Commands, events, wire format, and evolution

An event body is `{ field[:Type] â€¦ }` or `= fields(Command)`. Version defaults to 1. `vN` with
`N > 1` requires a contiguous `upcast from v(N-1) = HOLE`.

Retirement is a **two-stage protocol** with two mutually exclusive markers:

- `retiring event` â€” retirement has started; the event must keep at least one **live** emitting
  transition while operators terminalize or truncate affected streams.
- `deprecated event` â€” off the live write path (no *live* transition may emit it) but still
  decodable. A **replay-only** emitting transition may, and generally must, remain: without one,
  `DeprecatedEventReplayHazard` warns that stored history is no longer replayable.

<Callout type="warn">
`deprecated` alone protects only the codec. Hydration still needs an edge that inverts the event â€”
see [Retire an event type](/docs/keiro/how-to/retire-an-event-type).
</Callout>

An aggregate permits at most one `wire`, `projection`, and `snapshot`. The wire clause captures event
kind, field naming, and schema version; changes are diffed as persisted decode changes.

### Generated and hand-owned files

Generated: `Domain`, `Codec`, `EventStream`, `Projection`, and `Harness`. Hand-owned: `Holes`, which
contains the symbolic transition, projection SQL, and upcaster bodies. The public generated stream is
validated with `mkEventStreamOrThrow`; never replace it with the unchecked constructor.

Scaffolding also emits **one context-wide** `Generated.<Context>.ReplayAudit` module carrying
`auditTargets :: [SomeAuditTarget]` â€” every aggregate, in declaration order, each validating
discovered stream names against its generated category. Feed it to
[`Keiro.ReplayAudit`](/docs/keiro/reference/replay-audit).

Generated snapshot codecs compose a **fold fingerprint** into `stateShapeHash` via
`withFoldFingerprint`. It is a deterministic digest of the spec-visible replay surface â€” state order
and terminality, register types and initials, transition modes, guards, writes, emits, targets, and
transitively referenced rule bodies â€” so a spec-visible fold change invalidates old snapshots
automatically and raises the non-breaking `AggFoldSurfaceChanged` advisory at diff time. Logic changed
only inside `Holes` is *not* covered: bump `stateCodecVersion` yourself.

### Principal validation rules

References, states, commands, events, register writes, initial values, guard atoms, reachability,
terminal states, event versions/upcasters, deprecated writes, replay-only discipline, identifiers,
duplicate names, and wire/schema coherence are all checked. Common codes include `UndeclaredCommand`,
`WriteTargetNotRegister`, `RegisterInitialOutOfScope`, `GuardAtomOutOfScope`, `UnreachableState`,
`TerminalHasOutgoing`, `ClockSampled`, `EvtVersionMissingUpcaster`, `DuplicateUpcasterSource`,
`UpcasterChainGap`, `DeprecatedEventStillEmitted`, `DeprecatedEventReplayHazard`,
`EventRetirementInProgress`, `ReplayOnlyEmitsNothing`, and `ReplayOnlyCommandStillLive`.

## Projection

```keiro-dsl
projection transfer_decisions key=reservationId
  status-map {
    TransferReservationCreated=>held
    TransferReservationConfirmed=>confirmed
  }
```

`consistency=Strong|Eventual` is accepted as legacy syntax; a same-named first-class read model owns
the actual mode. A total status map must name every emitted event exactly. Use
`status-map partial { â€¦ }` only when omitted events intentionally do not change projected status.
Every included key must still name one declared event, without suffix matching or duplicates.

An inline read model must be referenced by a same-named aggregate projection. Generated projection
holes use the schema-qualified table constant from the first-class node.

## Snapshot

```keiro-dsl
snapshot every 100
  state-codec version=1 shape-hash="<captured-live-hash>"
```

The alternate policy is `snapshot on-terminal`. `every` must be at least 1; codec version must be at
least 1; the captured hash must be non-empty. Scaffold lowers the policies to `Every n` or
`OnTerminal` and `defaultStateCodec version`.

The hash is the live module-qualified register/state shape, not the event wire hash. Start with a
placeholder, scaffold, run `keiro-dsl-conformance-snapshot`, copy the printed live hash, regenerate,
and rerun. A mismatch is a cache miss/replay concern, not a reason to decode incompatible state.
Custom snapshot predicates are hand-owned and intentionally absent from the notation.

## Read model

```keiro-dsl
readmodel transfer_decisions {
  table = "transfer_decisions"
  schema = "hospital_capacity"
  columns {
    reservation_id text required
    hospital_id text required
    status text required
    decided_at timestamptz
  }
  version = 1
  shape = "fnv1a:3717f6d9e3c44bd6"
  consistency = Strong
  scope = category "reservation"
  feed = subscription
  subscription = "hospital-capacity-transfer-decisions-sub"
}
```

### Closed values and defaults

| Clause | Values/default |
|---|---|
| column type | `text`, `int`, `bigint`, `bool`, `timestamptz`, `jsonb`, `numeric` |
| nullability | omitted = nullable; `required` = not null |
| consistency | `Strong` or `Eventual` |
| scope | omitted = `entire-log`; `entire-log` or `category "name"`, only with `Strong` |
| feed | `inline` or `subscription` |
| subscription | omitted = `<context>-<node-with-hyphens>-sub` |

The registry name is `<context>-<node_name_with_hyphens>`. Shape is FNV-1a-64 over the table and
ordered `name:type:req|null` column surface. `check` prints the expected fixture on drift; changing
the declared shape also requires a version bump.

`Strong` requires `feed = subscription`; an inline-only model has no cursor for a strong read to
await. `scope` is meaningful only for `Strong`. `feed = inline` must be owned by an aggregate
projection. Query operations, routers, and dispatches resolve the node and its declared columns.

Generated: `ReadModelTable`, `ReadModel`, `ReadModelHarness`. Hand-owned: `ReadModelHoles` for inline
apply, async apply, and query behavior. SQL must interpolate the generated qualified-table constant,
not depend on `search_path`.

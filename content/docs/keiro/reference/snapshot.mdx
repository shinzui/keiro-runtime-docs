The types and functions on this page span `Keiro.Snapshot`, `Keiro.Snapshot.Codec`,
`Keiro.Snapshot.Policy` (in `keiro-core`), and `Keiro.Snapshot.Schema`. A snapshot is a disposable,
advisory cache of folded aggregate state. The event log remains durable truth.

## Lookup results and hydration seeds

```haskell
data SnapshotSeed rs s = SnapshotSeed
  { state :: !s
  , registers :: !(RegFile rs)
  , streamVersion :: !StreamVersion
  }

data SnapshotMissReason
  = SnapshotNoStream
  | SnapshotNotFound
  | SnapshotDecodeFailed !Text

data SnapshotLookup rs s
  = SnapshotUnavailable !SnapshotMissReason
  | SnapshotHit !(SnapshotSeed rs s)

lookupSnapshotSeed ::
  (Store :> es) =>
  StreamName ->
  StateCodec (s, RegFile rs) ->
  Eff es (SnapshotLookup rs s)

hydrateWithSnapshot ::
  (Store :> es) =>
  StreamName ->
  StateCodec (s, RegFile rs) ->
  Eff es (Maybe (SnapshotSeed rs s))
```

`SnapshotNoStream` means Kiroku has no stream id for the name. `SnapshotNotFound` means the stream
exists but no row matches both the running codec version and shape hash; a missing row and an
incompatible row intentionally share this outcome. `SnapshotDecodeFailed message` means a compatible
row existed but its JSON did not decode. `SnapshotHit seed` returns the joint state/register pair and
the version after which hydration reads.

`hydrateWithSnapshot` is the compatibility helper that collapses every unavailable result to
`Nothing`. The command runner uses `lookupSnapshotSeed` so it can record a read hit or miss and count
decode failures separately. Every unavailable result falls back to full event replay.

## Encoding and writing

```haskell
encodeSnapshotStrict ::
  StateCodec state -> state -> IO (Either ErrorCall Value)

writeSnapshotEncoded ::
  (Store :> es) =>
  StreamId -> StreamVersion -> StateCodec state -> Value -> Eff es ()

writeSnapshot ::
  (Store :> es) =>
  StreamId -> StreamVersion -> StateCodec state -> state -> Eff es ()
```

`encodeSnapshotStrict` forces the complete JSON value. It returns an `ErrorCall` from a partial
encoder or an uninitialized keiki register; other exception types remain visible. The production
post-commit path calls it before `writeSnapshotEncoded`, so a lazy thunk cannot fail later while the
store consumes it. `writeSnapshot` preserves the older lazy helper contract; new advisory
post-commit code should use strict encode plus `writeSnapshotEncoded`.

Stream validation also forces the codec over `(initialState, initialRegisters)`. A snapshot-enabled
stream whose initial register file still contains an `uninit: <slot>` thunk is rejected by
`mkEventStream` before it can accept commands.

## Default codec and compatibility gates

```haskell
defaultStateCodec ::
  forall rs s.
  (FromJSON s, KnownRegFileShape rs, RegFileToJSON rs, ToJSON s) =>
  Int -> StateCodec (s, RegFile rs)
```

The default codec writes JSON shaped as `{ "state": ..., "registers": ... }`. Its caller-supplied
`Int` becomes `stateCodecVersion`; its `shapeHash` comes from keiki's stable register-file shape.
A stored snapshot is eligible only when **both** values match the running codec. Bump the explicit
version when state encoding changes in a way the register shape cannot capture.

Keiki 0.2 intentionally changed every non-empty 0.1 register shape hash once by replacing
module-qualified built-in names with pinned stable names. On upgrade, let the old row miss, replay
events from the beginning, and write a fresh snapshot. The empty `regfile:0` hash is unchanged. Never
relabel old snapshot bytes with a new hash.

## Policy

`SnapshotPolicy` decides when the post-command folded pair is cached:

```haskell
data SnapshotPolicy state
  = Never
  | Every !Int
  | OnTerminal
  | Custom !(Terminality -> state -> StreamVersion -> Bool)

shouldSnapshot ::
  SnapshotPolicy state -> Terminality -> state -> StreamVersion -> Bool

shouldSnapshotSpan ::
  SnapshotPolicy state -> Terminality -> state ->
  StreamVersion -> StreamVersion -> Bool
```

The command path uses `shouldSnapshotSpan` over `(preVersion, postVersion]`, so a multi-event append
that crosses a positive `Every n` boundary fires even if its final version is not itself a multiple.
`OnTerminal` and `Custom` see the post-append state and terminality. `stateCodec = Nothing` disables
snapshotting; stream validation rejects that setting when the policy is not `Never`.

## Storage and version replacement

```haskell
lookupSnapshot ::
  (Store :> es) => StreamId -> Int -> Text -> Eff es (Maybe SnapshotRow)

writeSnapshotRow ::
  (Store :> es) => SnapshotWrite -> Eff es ()
```

`keiro.keiro_snapshots` holds one row per stream, with `state`, `stream_version`,
`state_codec_version`, `regfile_shape_hash`, and timestamps. Lookup filters on stream id, codec
version, and shape hash before decoding.

The upsert's version rule is conditional:

- With the **same** codec version and shape hash, a lower stream version is ignored. Late work cannot
  regress a compatible snapshot.
- With a **different** codec version or shape hash, the row may be replaced even at a lower stream
  version. That permits a rolled-back deployment to reclaim the single snapshot slot.

During a mixed-version rollout, incompatible writers can therefore replace each other's rows and
cause repeated misses/full replays. That is performance thrash, not state corruption. Avoid a long
mixed window and monitor snapshot misses.

## Caller-visible and operational outcomes

All command-path snapshot work occurs after the event append has committed, except validation of the
initial state at stream construction.

| Condition | Event append committed? | Caller-visible result | Fallback / next step | Metric with `KeiroMetrics` |
|---|---:|---|---|---|
| No stream, no compatible row, version mismatch, or shape mismatch | Not applicable during lookup | Hydration continues | Full event replay | `keiro.snapshot.read.misses` |
| Compatible row does not decode | Not applicable during lookup | Hydration continues | Full event replay; inspect or discard the row | `keiro.snapshot.read.misses` + `keiro.snapshot.decode.failures` |
| Snapshot-seeded tail replay fails | No new append yet | Seed is abandoned | Full replay determines the authoritative result | Lookup was already counted as a hit; no separate seed-tail counter |
| Just-appended batch cannot replay | **Yes** | Successful `CommandResult` | Stop further writes and investigate the poisoned stream | `keiro.snapshot.apply.divergence`; span attribute `keiro.replay.divergence` |
| Post-commit snapshot encode raises `ErrorCall` | **Yes** | Successful `CommandResult` | No snapshot written; a later command may try again | `keiro.snapshot.encode.failures` |
| Post-commit snapshot store write fails | **Yes** | Successful `CommandResult` | Existing row remains; a later command may try again | `keiro.snapshot.write.failures` |
| Initial snapshot encode fails during `mkEventStream` | No | `Left [EventStreamWarning]` | Initialize all persisted registers or fix the codec before startup | None |

`verifyReplayOnAppend` defaults to `True`. Setting it to `False` skips the post-append replay only for
a stream without a state codec; snapshot-enabled streams always need the fold to compute snapshot
state.

See [Add a snapshot](/docs/keiro/how-to/add-a-snapshot) for setup and recovery, and
[Consistency and snapshots](/docs/keiro/explanation/consistency-and-snapshots) for why these failures
are advisory rather than alternate sources of truth.

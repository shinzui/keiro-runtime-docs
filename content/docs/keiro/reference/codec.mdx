## `Codec`

Everything a stream needs to serialize, deserialize, and migrate its events. It is a **record value**,
not a typeclass.

## `Upcaster`

One rung of an upcaster chain: the source version it upgrades **from**, and a pure migration.

```haskell
type Upcaster = (Int, EventType -> Value -> Either Text Value)
```

A rung receives the **stored** `EventType` alongside the payload, so one codec can own several event
kinds without guessing the kind from payload fields. That is how `keiro-dsl` lowers several
same-version event bumps into one rung: the generated dispatcher selects the event-specific upcaster
by tag and passes every other kind through byte-for-byte.

<Callout type="warn">
Upcasters are **permanent compatibility code**. A rung lives while any stored payload at its source
version persists — which, for an append-only log without truncation, means forever. Keiro has no
mechanism for declaring an old stored version unsupported.
</Callout>

<Callout type="warn">
Use `mkCodec` for production codecs. It validates `schemaVersion >= 1`, duplicate event tags,
duplicate upcaster sources, out-of-range upcaster sources, and missing rungs before runtime. The raw
record constructor remains exported for low-level tests and escape hatches.

You get this check whether or not you call `mkCodec` yourself: `validateEventStreamWith` runs it at
the stream boundary, so `mkEventStream`, `mkEventStreamWith`, and `mkEventStreamOrThrow` all fail
construction on a misconfigured codec. Only `mkEventStreamUnchecked` bypasses it.
</Callout>

```haskell
mkCodec :: Codec e -> Either CodecConfigError (Codec e)

data CodecConfigError
  = CodecSchemaVersionInvalid !Int
  | CodecDuplicateEventTypes ![EventType]
  | CodecDuplicateUpcasterSources ![Int]
  | CodecUpcasterSourceOutOfRange !Int !Int
  | CodecUpcasterChainIncomplete ![Int] !Int
  deriving stock (Generic, Eq, Show)
```

`CodecUpcasterChainIncomplete missing target` means the codec says its current `schemaVersion` is
`target`, but one or more source-version rungs in `[1 .. target - 1]` are absent. That turns
"we will find out during hydration" into an explicit construction failure.

## `CodecError`

```haskell
data CodecError
  = UnknownEventType !EventType ![EventType]
  | InvalidSchemaVersion !Int
  | UnknownVersion !Int
  | VersionAhead !Int !Int
  | UpcasterError !Int !Text
  | DecodeFailed !Text
  | GapInUpcasterChain !Int !Int
  | IncompleteUpcasterChain !Int !Int
  | MalformedSchemaVersionStamp !Value
  | NonObjectCallerMetadata !Value
  deriving stock (Generic, Eq, Show)
```

<Callout type="warn">
An unknown `eventType` on read is **fatal** (`UnknownEventType`), carrying the offending tag and the
allowed set. There is no "skip unknown events" mode.
</Callout>

## Functions

```haskell
encodeForAppend             :: Codec e -> e -> Either CodecError EventData
encodeForAppendWithMetadata :: Codec e -> Maybe Value -> e -> Either CodecError EventData
decodeRecorded              :: Codec e -> RecordedEvent -> Either CodecError e
decodeRaw                   :: Codec e -> EventType -> Int -> Value -> Either CodecError e
migrateToCurrent            :: Codec e -> EventType -> Int -> Value -> Either CodecError Value
extractSchemaVersion        :: RecordedEvent -> Either CodecError Int
metadataFor                 :: Int -> Maybe Value -> Either CodecError Value
```

Behavior to know:

- `encodeForAppendWithMetadata` stamps `schemaVersion` into fresh or supplied metadata via
  `metadataFor`, and the **schema-version key always wins** over any clashing caller key, so the
  version on disk is authoritative. Caller metadata must be a JSON object; non-object metadata is
  rejected with `NonObjectCallerMetadata`.
- `decodeRecorded` reads the stamped version (`extractSchemaVersion`, defaulting to `1` only when
  metadata is absent or lacks the key), runs `migrateToCurrent` (replay upcasters `n, n+1, …` up to
  `schemaVersion`), then the current event-type-aware `decode`. A present malformed stamp is
  `MalformedSchemaVersionStamp`, not a silent default.
- A stored payload with a source version greater than the codec's current `schemaVersion` fails with
  `VersionAhead`; a chain that simply ends before the target fails with `IncompleteUpcasterChain`.

The worked example is the `jitsurei` order codec (`orderCodec`, `schemaVersion = 2`,
`upcasters = [(1, const upcastOrderPlacedV1)]` — the `const` discards the `EventType` because that
codec migrates a single kind at this rung); see
[Evolve an event schema](/docs/keiro/how-to/evolve-an-event-schema).

<Callout type="warn">
`schemaVersion` is **aggregate-global**, not per event kind. When a second event kind changes, assign
it `aggregate-max + 1`, not that event's private previous version plus one. That keeps every payload
stamp aligned with the one aggregate-wide chain.
</Callout>

A decode golden proves an old payload still *decodes*. It never proves that the decoded event still
has an inverting edge or folds to the same state — that is a separate question answered by
[the replay audit](/docs/keiro/reference/replay-audit). See
[Evolution and replayability](/docs/keiro/explanation/evolution-and-replayability) for how the two
gates divide the work, and
[Deploy ordering](/docs/keiro/reference/deploy-ordering) for why a codec bump cannot ride an ordinary
rolling deploy.

`EventStream` (and its `SnapshotPolicy`/`StateCodec`) live in `Keiro.EventStream`; the `Stream`
handle lives in `Keiro.Stream`. Both are re-exported from `Keiro`.

## `EventStream`

The complete description of one persistent event stream: a keiki transducer plus everything keiro
needs to run it against a durable store. The type parameters thread through from the transducer —
`phi` = guard alphabet, `rs` = register set, `s` = control state, `ci` = command input, `co` = event
output.

<Callout type="info">
`stateCodec = Nothing` disables snapshotting **regardless** of `snapshotPolicy`. The `jitsurei` order
stream shows both ends: `orderEventStream` uses `snapshotPolicy = Never`, `stateCodec = Nothing`,
while `snapshotOrderEventStream` uses `snapshotPolicy = Every 2`,
`stateCodec = Just (defaultStateCodec @OrderRegs @OrderState 1)`.
</Callout>

## Replay-safety validation (`Keiro.EventStream.Validate`)

An `EventStream` is only as safe to replay as the keiki transducer inside it.
`Keiro.EventStream.Validate` lifts keiki's pure umbrella check (`validateTransducer`, keiki EP-56)
to the `EventStream` boundary, so a service can assert every stream is sound **before** it is ever
hydrated — with no SMT solver. keiki can prove statically that a transducer is *replay-safe* (every
command field an edge consumes is recoverable from the event(s) it emits), that its guards are
deterministic, and that no edge is dead.

```haskell
data EventStreamWarning = EventStreamWarning
  { eswStreamLabel :: !Text   -- the caller-supplied stream label
  , eswReason      :: !Text   -- rendered from the keiki warning
  }
  deriving stock (Eq, Show)

validateEventStream ::
  (Bounded s, Enum s, Ord s, Show s) =>
  Text -> EventStream (HsPred rs ci) rs s ci co -> [EventStreamWarning]

validateEventStreamWith ::
  (Bounded s, Enum s, Ord s, Show s) =>
  ValidationOptions -> Text -> EventStream (HsPred rs ci) rs s ci co -> [EventStreamWarning]

mkEventStream ::
  (Bounded s, Enum s, Ord s, Show s) =>
  Text -> EventStream (HsPred rs ci) rs s ci co ->
  Either [EventStreamWarning] (EventStream (HsPred rs ci) rs s ci co)
```

- `validateEventStream label es` runs the check with `defaultValidationOptions` and returns the
  labelled warnings; an **empty list** means the stream passed every enabled check.
- `validateEventStreamWith opts label es` is the same with caller-chosen `ValidationOptions` (e.g.
  to narrow the checks for a stream with a known-benign warning).
- `mkEventStream label es` is the fail-fast smart constructor: `Left warnings` for an unsafe stream,
  `Right es` for a sound one. The bare `EventStream { … }` record literal stays available for
  low-level callers who do not want the check.

The `label` is echoed into every `EventStreamWarning` so a multi-aggregate service can tell *which*
stream is at fault; `eswReason` renders the underlying keiki warning (`hidden-input @…`,
`nondeterministic @…`, `possibly-dead @…`, or `opaque-guard @…`, each tagged with the source
vertex). The check requires the control state `s` to satisfy `(Bounded s, Enum s, Ord s, Show s)` —
it walks `[minBound .. maxBound]` exactly as the renderers and `defaultStateCodec` shape hash do —
and the transducer must use keiki's standard `HsPred` guard carrier.

The enabled checks come from keiki's `ValidationOptions` (carried through `validateTransducer`):

<TypeTable
  type={{

    checkDeterminism: { type: "Bool", description: "Run the (pure, structural) determinism check on guards.", default: "True" },

<Callout type="info">
This is a **pure, build-time** check — no solver, no database. Run it in a test, or at service
start-up over your registry of streams, so a replay-unsafe transducer fails fast instead of
silently corrupting hydration later. For *why* replay-safety matters, see [Why SymTransducer, not
Decider](/docs/keiro/explanation/why-symtransducer-not-decider). To visualize the same transducer as
a diagram, see keiki's [render a mermaid diagram](/docs/keiki/how-to/render-a-mermaid-diagram).
</Callout>

## `SnapshotPolicy`

When keiro writes a snapshot of the folded `(state, registers)`.

```haskell
data SnapshotPolicy state
  = Never                                  -- never snapshot; always replay the full log
  | Every !Int                             -- snapshot when version is a positive multiple of n
  | OnTerminal                             -- snapshot only at a final state
  | Custom !(state -> StreamVersion -> Bool)
  deriving stock (Generic)
```

## `Stream` — the phantom-typed handle

A `StreamName` tagged with a phantom type `a` identifying which aggregate it names, so a name for one
aggregate cannot be passed where another is expected.

```haskell
newtype Stream a = Stream { name :: StreamName }
  deriving stock (Generic, Eq, Ord, Show)

stream        :: Text -> Stream a                                   -- build a handle from a raw stream-name Text
streamName    :: Stream a -> StreamName                              -- recover the underlying StreamName
mapStreamName :: (StreamName -> StreamName) -> Stream a -> Stream a  -- transform the name, keep the tag
```

<Callout type="info">
The design notes' old name for this type was `AggregateId a`; the shipped type is `Stream a`. Trust
the source.
</Callout>

## Safe, category-based construction (`StreamCategory`)

`stream` takes a raw `Text`, so the physical name `"order-" <> orderIdText orderId` is hand-assembled
at every call site. That leaks an invariant the **store**, not the caller, owns: in kiroku a stream
name's *category* is the substring before its first `-`, and category-scoped reads
([`Kiroku.Store.Read.readCategory`](/docs/kiroku/reference/core-types)) and category subscriptions
fan in every stream that shares that prefix. So the literal `"order-"` is not cosmetic — it is the
**join key** deciding which streams a projection or subscription sees. Hand-writing it per call site
has three hazards: no single source of truth (a typo spawns an orphan category), silent
mis-parsing (a "category" that itself contains a `-`, e.g. a saga prefix `"hospital-surge-"`, parses
as the category `hospital`, fanning the saga's stream into the wrong aggregate), and no typed bridge
back to the `CategoryName` a `readCategory`/subscription needs.

`Keiro.Stream` closes this with a validated, phantom-typed **category**: declare it once and derive
both per-entity streams and the category name from it.

```haskell
newtype StreamCategory a = StreamCategory { categoryTextOf :: Text }
  deriving stock (Generic, Eq, Ord, Show)

data CategoryError
  = CategoryEmpty                    -- the empty string
  | CategoryContainsSeparator !Text  -- contains the reserved `-` category/id boundary
  | CategoryReserved !Text           -- equals a store-reserved name ($all)
  deriving stock (Eq, Show, Generic)

category       :: Text -> Either CategoryError (StreamCategory a)  -- validating smart constructor
categoryUnsafe :: Text -> StreamCategory a                         -- partial; `error`s on an invalid literal
categoryName   :: StreamCategory a -> CategoryName                 -- the matching CategoryName for category reads/subs
entityStream   :: StreamCategory a -> Text -> Stream a             -- render "<category>-<id>"
entityStreamId :: StreamIdSegment i => StreamCategory a -> i -> Stream a

class StreamIdSegment i where
  renderIdSegment :: i -> Text
-- instances for Text and String ship; add one per typed id:
-- instance StreamIdSegment OrderId where renderIdSegment = orderIdText
```

- `category t` **rejects** the empty string, any text containing `-` (kiroku's category/id boundary —
  this is hazard #2 turned into a fail-stop), and the reserved name `$all`. Returns
  `Left CategoryError` otherwise.
- `categoryUnsafe t` is for static, known-good literals at a definition site
  (`orderCategory = categoryUnsafe "order"`); it `error`s on an invalid category, so **never** pass
  user input to it.
- `entityStream`/`entityStreamId` delegate the name mechanics to kiroku's `streamNameInCategory`, so
  the category rule stays single-sourced in the store. The phantom type carries from the
  `StreamCategory`, so the result is correctly tagged.
- `categoryName cat` recovers the `CategoryName` for category-scoped reads and subscription targets
  **from the same definition** — and by construction `categoryName cat` always equals
  `Kiroku.Store.Types.categoryName (streamName (entityStream cat id))`, so the per-entity stream and
  the category read can never disagree.

```haskell
orderCategory :: StreamCategory a
orderCategory = Stream.categoryUnsafe "order"          -- validated once at the definition site

orderStream :: OrderId -> Stream OrderEventStream
orderStream = Stream.entityStream orderCategory . orderIdText   -- renders "order-<id>"

orderCommandStream :: OrderId -> Stream OrderCommand
orderCommandStream = Stream.entityStream orderCategory . orderIdText
```

This is exactly how the `jitsurei` order stream now builds its handles — the two `stream ("order-" <>
…)` literals collapse to one declared category.

<Callout type="warn">
The type is named `StreamCategory`, **not** `Category`, to avoid clashing with the
`Kiroku.Store.Subscription.Types.Category` subscription-target constructor that consumer code often
imports alongside this module. Join a **compound** category with `_` (e.g. `"hospital_surge"` for a
saga over hospital surges) — never with `-`, which `category` rejects. The `:` separator is reserved
for the workflow stream family (`wf:<name>`).
</Callout>

For why an `EventStream` carries a `SymTransducer` directly rather than a decide/evolve pair, see
[Why SymTransducer, not Decider](/docs/keiro/explanation/why-symtransducer-not-decider).

**keiro** (経路, "route / path") is a Haskell **library you import** — not a server you operate —
that composes three lower libraries into one event-sourcing and workflow framework:
**kiroku** (記録, the append-only PostgreSQL event store), **keiki** (the pure symbolic-register
finite-state transducer that is the decision core), and **shibuya** (the supervised
subscription/worker substrate). Its thesis: aggregates, process managers, and sagas are all the
*same* mathematical object — keiki's `SymTransducer phi rs s ci co` — persisted to *one* substrate,
your Postgres. On top of that v1 substrate, the **v2 durable-execution runtime** (`Keiro.Workflow`)
adds journaled named-step workflows for processes that read as one long-running function with in-line
waits.

<Callout type="warn">
  keiro is in **active development** (`0.1.0.0`). APIs may change; pin a version. The v2
  durable-execution workflow engine (`Keiro.Workflow`) has **shipped** — see
  [Understanding durable execution](/docs/keiro/explanation/durable-execution) — now including
  continue-as-new journal rotation (`continueAsNew`) and the `patch` versioning API, plus
  [LISTEN/NOTIFY push delivery and consumer-group sharding](/docs/keiro/explanation/scaling-the-workers)
  for scaling the background workers. A new authoring toolchain,
  [keiro-dsl](/docs/keiro/explanation/the-keiro-dsl-toolchain), lets you specify a whole service as a
  typed `.keiro` file and scaffold its deterministic layer.
</Callout>

## How it fits together

A command runs through keiro's **Hydrate → Transduce → Append** cycle: `runCommand` reads the
target stream into state (Hydrate), runs the keiki transducer to turn the command into events
(Transduce), and appends them to kiroku under optimistic concurrency (Append), retrying on conflict.

```mermaid
flowchart LR
  Cmd["Your command (ci)"]
  RC["runCommand · Keiro.Command"]
  Keiki["keiki transducer · Transduce"]
  Kiroku[("kiroku · PostgreSQL event store")]
  Shibuya["shibuya · subscriptions / workers"]
  Read["Your read models / process managers"]

  Cmd --> RC
  RC -->|"Hydrate: readStreamForward"| Kiroku
  RC -->|"Transduce"| Keiki
  Keiki -->|"events (co)"| RC
  RC -->|"Append: appendToStream (ExpectedVersion)"| Kiroku
  Kiroku -->|"subscribe"| Shibuya
  Shibuya --> Read
```

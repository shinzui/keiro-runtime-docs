`jitsurei` (実例, "worked example") is the older end-to-end example package that
many Keiro pages use as a shared domain vocabulary. Its order-fulfillment and
incident-escalation scenarios made command handling, read models, snapshots,
process managers, routers, and durable timers concrete without inventing a new
domain on every page.

<Callout type="warn">
  `keiro-runtime-jitsurei` has **not** been modernized for the July 2026 release
  set. Its historical source paths and command names remain useful as
  architecture annotations, but its build and scenarios are not proof of Keiro
  0.3, keiki 0.2, Kiroku 0.3, Shibuya 0.8, pgmq-hs 0.4, or pg-migrate 1.1
  compatibility. Do not use the old `just jitsurei-*` or `cabal run jitsurei`
  commands as an announcement release gate.
</Callout>

## What the legacy example covered

| Area | Historical source anchor | Current documentation |
| --- | --- | --- |
| Order command cycle and codec upcasting | `Jitsurei.Domain`, `Jitsurei.OrderStream` | [Get started with Keiro](/docs/keiro/tutorials/getting-started), [Codec and schema evolution](/docs/keiro/explanation/codec-and-schema-evolution) |
| Inline read models and snapshots | `Jitsurei.ReadModels`, `Jitsurei.Snapshots` | [Your first read model](/docs/keiro/tutorials/your-first-read-model), [Add a snapshot](/docs/keiro/how-to/add-a-snapshot) |
| Process managers and durable timers | `Jitsurei.FulfillmentProcess`, `Jitsurei.EscalationProcess`, `Jitsurei.Timers` | [Your first process manager](/docs/keiro/tutorials/your-first-process-manager), [Drive the timer worker](/docs/keiro/how-to/drive-the-timer-worker) |
| Content-based routing | `Jitsurei.Paging`, `Jitsurei.AgentQualRouter` | [Routers and content-based dispatch](/docs/keiro/explanation/routers-and-content-based-dispatch) |
| Inbox and outbox | No runnable Kafka scenario was included | [Inbox pattern](/docs/keiro/explanation/the-inbox-pattern), [Outbox pattern](/docs/keiro/explanation/the-outbox-pattern) |

The module names still appear in explanation and walkthrough pages where they
make a source-level relationship easier to discuss. Read them as historical
illustrations of the architecture, not as copy-ready current package imports.

## The `Codec` record and `CodecError`

A codec is a plain **value-level record**, not a typeclass:

```haskell
-- keiro-core/src/Keiro/Codec.hs
type Upcaster = (Int, Value -> Either Text Value)  -- (source version, pure migration)

data Codec e = Codec
  { eventTypes :: !(NonEmpty Text)          -- the complete event-type allow-list this codec owns
  , eventType :: !(e -> Text)               -- project a domain value to its wire tag
  , schemaVersion :: !Int                   -- current payload version; must be >= 1
  , encode :: !(e -> Value)                 -- current-version JSON serialization
  , decode :: !(Value -> Either Text e)     -- only ever sees payloads migrated to schemaVersion
  , upcasters :: ![Upcaster]                -- migrations keyed by source version
  }
  deriving stock (Generic)
```

    eventType: { type: "e -> Text", description: "Projects a domain value to its wire tag; must land in eventTypes." },
    schemaVersion: { type: "Int", description: "The current payload version; must be >= 1. Stamped on append, the migration target on read." },

Because it is a value, one event type can carry several codecs (different streams, different policies),
and upcasters are first-class data you can list, test, and compose. The errors it can raise are a
closed set of six:

```haskell
data CodecError
  = UnknownEventType !EventType ![Text]   -- tag not in eventTypes (offending tag, allowed set)
  | InvalidSchemaVersion !Int             -- schemaVersion is not >= 1
  | UnknownVersion !Int                   -- stored version < 1, or beyond the chain
  | UpcasterError !Int !Text              -- an upcaster rejected its input (source version, msg)
  | DecodeFailed !Text                    -- the current decode rejected a migrated payload
  | GapInUpcasterChain !Int !Int          -- reached version n, next rung starts later
  deriving stock (Generic, Eq, Show)
```

Each constructor is raised at a specific spot below: `InvalidSchemaVersion` and `UnknownEventType` on
the write side (and `UnknownEventType` again on read for an unknown tag); `UnknownVersion`,
`UpcasterError`, `GapInUpcasterChain` inside migration; and `DecodeFailed` from the final decode. The
command cycle wraps these as `EncodeFailed` (on write) or `HydrationDecodeFailed` (on hydrate), the two
codec-shaped `CommandError`s from [chapter 01](/docs/keiro/walkthrough/command-cycle/01-command-types-and-errors).

## Write: stamp the schema version

```haskell
encodeForAppend codec value = encodeForAppendWithMetadata codec Nothing value

encodeForAppendWithMetadata codec metadata value = do
  unless (codec ^. #schemaVersion > 0) $ Left (InvalidSchemaVersion (codec ^. #schemaVersion))
  let selectedType = codec ^. #eventType $ value
  unless (selectedType `List.elem` NonEmpty.toList (codec ^. #eventTypes)) $
    Left (UnknownEventType (EventType selectedType) (NonEmpty.toList (codec ^. #eventTypes)))
  pure EventData { eventId = Nothing, eventType = EventType selectedType
                 , payload = codec ^. #encode $ value
                 , metadata = Just (metadataFor (codec ^. #schemaVersion) metadata)
                 , causationId = Nothing, correlationId = Nothing }

metadataFor version existing =
  Object $ baseObject existing & KeyMap.insert schemaVersionKey (Number (fromIntegral version))
  where baseObject (Just (Object object)) = object
        baseObject _ = KeyMap.empty
```

`encodeForAppend` is `encodeForAppendWithMetadata codec Nothing`. The general form guards two ways
before encoding: `InvalidSchemaVersion` if the codec is misconfigured (`schemaVersion` not `> 0`), and
`UnknownEventType` if the value's tag is outside `eventTypes`. Then it stamps the version through
`metadataFor`, which `insert`s the `schemaVersion` key into the (possibly caller-supplied) metadata
object.

<Callout type="warn">
The schema-version key is `insert`ed **last**, so it always wins over any clashing key in the caller's
metadata. The version recorded on disk is authoritative and cannot be accidentally overwritten â€” which
is exactly why the [`metadata` option](/docs/keiro/walkthrough/command-cycle/01-command-types-and-errors)
on `RunCommandOptions` can be merged freely without endangering the stamp.
</Callout>

## Read: check the tag, migrate, decode

```haskell
decodeRecorded codec recorded = do
  unless (isKnownEventType (recorded ^. #eventType) codec) $
    Left (UnknownEventType (recorded ^. #eventType) (NonEmpty.toList (codec ^. #eventTypes)))
  decodeRaw codec (extractSchemaVersion recorded) (recorded ^. #payload)

decodeRaw codec version payload = do
  migrated <- migrateToCurrent codec version payload
  case codec ^. #decode $ migrated of
    Right value -> Right value
    Left message -> Left (DecodeFailed message)

-- Defaults to 1 when metadata is absent, not an object, lacks the key, or is non-integer.
extractSchemaVersion recorded =
  fromMaybe 1 $ do
    Object object <- recorded ^. #metadata
    Number number <- KeyMap.lookup schemaVersionKey object
    Scientific.toBoundedInteger number
```

`decodeRecorded` is the read door. First it checks the stored tag against `eventTypes` â€”
`UnknownEventType` if it misses. Then `extractSchemaVersion` reads the stamped version back, defaulting
to `1` when the metadata is absent, not an object, missing the key, or non-integer (so pre-stamp events
still decode). `decodeRaw` migrates the payload up to the current version and runs the codec's `decode`,
lifting a rejection to `DecodeFailed`. The migration loop is the heart of it:

```haskell
migrateToCurrent codec sourceVersion payload
  | sourceVersion >= codec ^. #schemaVersion = Right payload
  | sourceVersion < 1 = Left (UnknownVersion sourceVersion)
  | otherwise = go sourceVersion payload
  where
    go version current
      | version >= codec ^. #schemaVersion = Right current
      | otherwise = case Prelude.lookup version (codec ^. #upcasters) of
          Nothing -> case nextChainStart version of
            Just nextVersion -> Left (GapInUpcasterChain version nextVersion)
            Nothing -> Left (UnknownVersion version)
          Just upcast -> case upcast current of
            Left message -> Left (UpcasterError version message)
            Right next -> go (version + 1) next
```

A payload already at or beyond `schemaVersion` is returned unchanged; a version below `1` is
`UnknownVersion`. Otherwise the loop looks up the rung keyed by the *current* version, applies it (a
`Left` becomes `UpcasterError`), and recurses one version higher. A missing rung is the subtle case: if
a *later* rung exists the chain has a hole, so it is `GapInUpcasterChain version nextVersion`; if none
exists at all it is `UnknownVersion`. The chain must therefore be ascending and contiguous.

<Callout type="warn">
There is no "skip unknown event" path. An `UnknownEventType` (or any other `CodecError`) on read aborts
the whole hydrate replay as `HydrationDecodeFailed` â€” the single decode error we traced in
[chapter 02](/docs/keiro/walkthrough/command-cycle/02-hydration). A codec must own *every* tag that has
ever been written to its stream.
</Callout>

## The jitsurei v1â†’v2 upcaster

The jitsurei order codec is the worked example. `orderCodec :: Codec OrderEvent` carries
`schemaVersion = 2`, `upcasters = [(1, upcastOrderPlacedV1)]`, and
`eventTypes = "OrderPlaced" :| ["PaymentApproved", "OrderPacked", "OrderShipped", "OrderCancelled"]`.
The single upcaster rewrites a v1 `OrderPlaced` (which used `"qty"` and an optional `"sku"`) into the v2
shape (required `"quantity"`, `"sku"` defaulted to `"UNKNOWN"`):

```haskell
upcastOrderPlacedV1 value =
  case parseEither parser value of
    Right migrated -> Right migrated
    Left message -> Left (Text.pack message)
  where
    parser = withObject "OrderPlacedV1" $ \objectValue -> do
      orderId  <- objectValue .: "orderId"
      sku      <- objectValue .:? "sku"
      quantity <- objectValue .: "qty"
      pure (object [ "kind" .= ("OrderPlaced" :: Text), "orderId" .= (orderId :: Text)
                   , "sku" .= maybe "UNKNOWN" id (sku :: Maybe Text), "quantity" .= (quantity :: Int) ])
```

Contrast the paging codec: `pageCodec` has `schemaVersion = 1` and `upcasters = []` â€” a codec with no
migration history, the common starting point. The cost of evolution is paid lazily on read: an old
event is migrated forward only when hydrated, never by rewriting the log. See
[Codec and schema evolution](/docs/keiro/explanation/codec-and-schema-evolution) for the conceptual
view and [Evolve an event schema](/docs/keiro/how-to/evolve-an-event-schema) for the recipe.

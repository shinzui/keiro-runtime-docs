This chapter reads `keiro-core/src/Keiro/Codec.hs` — the serialization boundary the command cycle
crosses on every write and every hydrate. Read
[02 — The transactional write path](/docs/keiro/walkthrough/command-cycle/02-the-transactional-write-path)
first.

## Write: stamp the schema version into metadata

The command runner encodes each emitted event with `encodeForAppendWithMetadata`, which delegates the
metadata stamp to `metadataFor`:

```haskell
-- keiro-core/src/Keiro/Codec.hs
metadataFor :: Int -> Maybe Value -> Value
metadataFor version existing =
  Object $
    baseObject existing
      & KeyMap.insert schemaVersionKey (Number (Prelude.fromIntegral version))
  where
    baseObject (Just (Object object)) = object
    baseObject _ = KeyMap.empty
```

The codec's `schemaVersion` is `insert`ed into the (possibly caller-supplied) metadata object. Because
the insert happens **last**, the schema-version key always wins over any clashing caller key — the
version recorded on disk is authoritative and cannot be accidentally overwritten.

## Read: check the tag, then migrate, then decode

```haskell
decodeRecorded :: Codec e -> RecordedEvent -> Either CodecError e
decodeRecorded codec recorded = do
  unless (isKnownEventType (recorded ^. #eventType) codec) $
    Left (UnknownEventType (recorded ^. #eventType) (NonEmpty.toList (codec ^. #eventTypes)))
  decodeRaw codec (extractSchemaVersion recorded) (recorded ^. #payload)
```

Two guards in three lines. First, the event tag must be in the codec's `eventTypes` allow-list; if not,
decoding **fails fatally** with `UnknownEventType` (carrying the offending tag and the allowed set).
There is no "skip unknown" path — during hydration a single unknown event aborts the whole replay.
Second, `extractSchemaVersion` reads the stamped version (defaulting to `1` when absent or invalid, so
pre-stamp events still decode), and `decodeRaw` runs `migrateToCurrent` — replaying the upcaster chain
from the stored version up to `schemaVersion` — before handing the migrated payload to the codec's
current `decode`.

<Callout type="warn">
The upcaster chain must be ascending and contiguous: reaching version `n` with the next available
rung starting later yields `GapInUpcasterChain n nextVersion`. A rung that returns `Left` becomes
`UpcasterError`, and a final `decode` that rejects the migrated payload becomes `DecodeFailed`.
</Callout>

Because this boundary sits on the hot hydrate path, the cost of evolution is paid lazily: an old event
is migrated forward only when it is read, never by rewriting the log. See
[Codec and schema evolution](/docs/keiro/explanation/codec-and-schema-evolution) for the conceptual
view and [Evolve an event schema](/docs/keiro/how-to/evolve-an-event-schema) for the recipe.

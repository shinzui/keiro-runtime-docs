This chapter reads `Keiro.Snapshot.Codec`, the supporting `StateCodec` definition, and the public
lookup/encode boundary in `Keiro.Snapshot`. Read
[00 — Start here](/docs/keiro/walkthrough/read-side/00-start-here) first.

## A snapshot must contain both halves

A keiki transducer carries a control state `s` and an independent typed register file `RegFile rs`.
The current command call site makes the pair explicit:

```haskell
Keiki.stepEither transducer (state current, registers current) command
```

Two streams may be at the same control vertex with different register values, so storing only `s`
would lose durable state. `defaultStateCodec` therefore targets `(s, RegFile rs)`:

```haskell
defaultStateCodec ::
  forall rs s.
  (FromJSON s, KnownRegFileShape rs, RegFileToJSON rs, ToJSON s) =>
  Int ->
  StateCodec (s, RegFile rs)

defaultStateCodec version = StateCodec
  { stateCodecVersion = version
  , shapeHash = regFileShapeHash (Proxy @rs)
  , encode = \(state, registers) ->
      object
        [ "state" Aeson..= state
        , "registers" Aeson..= regFileToJSON registers
        ]
  , decode = decodeSnapshotValue
  }
```

The state half uses its domain `ToJSON`/`FromJSON` instances. The register half uses keiki's strict
register-file JSON codec. Decoding either half can return `Left Text`; the whole snapshot then becomes
a cache miss rather than a partial seed.

## Initial state is validated, post-command state is forced

A JSON `Value` can hide a partial thunk. Keiro checks both moments at which that matters:

- `mkEventStream` forces a present codec over `(initialState, initialRegisters)`. It converts an
  `ErrorCall`, such as `uninit: lastAmount` from `emptyRegFile`, into a labelled
  `EventStreamWarning`. The service can refuse startup before accepting commands.
- After an append, `encodeSnapshotStrict codec finalState` evaluates and deeply forces the complete
  JSON value. It returns `Either ErrorCall Value`; other exception types deliberately remain visible.
  Only a successful value reaches `writeSnapshotEncoded`.

```haskell
encodeSnapshotStrict ::
  StateCodec state -> state -> IO (Either ErrorCall Value)
```

The post-command encode occurs after the event append has committed. A returned `ErrorCall` increments
`keiro.snapshot.encode.failures` and suppresses the cache write, but the command result remains
successful.

## Lookup retains the miss reason

The older `hydrateWithSnapshot` API returns `Maybe SnapshotSeed`. The command runner now uses the
more informative lookup:

```haskell
data SnapshotMissReason
  = SnapshotNoStream
  | SnapshotNotFound
  | SnapshotDecodeFailed !Text

data SnapshotLookup rs s
  = SnapshotUnavailable !SnapshotMissReason
  | SnapshotHit !(SnapshotSeed rs s)

lookupSnapshotSeed ::
  (Store :> es) =>
  StreamName ->
  StateCodec (s, RegFile rs) ->
  Eff es (SnapshotLookup rs s)
```

`SnapshotNoStream` means Kiroku has no stream id. `SnapshotNotFound` covers both a missing row and a
row that failed the version/shape compatibility filter. `SnapshotDecodeFailed` means a compatible
row existed but its JSON was unusable. All unavailable results trigger full event replay; the
command runner counts every one as `keiro.snapshot.read.misses` and counts decode failures again in
`keiro.snapshot.decode.failures`. A hit increments `keiro.snapshot.read.hits`.

`SnapshotSeed` contains the decoded state, registers, and stream version. Hydration resumes after
that version and carries the keiki replay wrapper across pages. If the tail fails, it abandons the
seed and retries from the aggregate's initial pair.

## Two compatibility discriminants

`StateCodec` carries:

```haskell
data StateCodec state = StateCodec
  { stateCodecVersion :: !Int
  , shapeHash :: !Text
  , encode :: !(state -> Value)
  , decode :: !(Value -> Either Text state)
  }
```

The row is eligible only when both `stateCodecVersion` and `shapeHash` match. The explicit version
catches state-encoding changes the register shape cannot see. The shape hash changes when the
type-level register layout changes, so incompatible register snapshots are rejected before decode.

Keiki 0.2 replaces module-qualified built-in type names with pinned stable shape names. As a result,
every non-empty 0.1 register shape hash changes once; the empty `regfile:0` hash does not. Keiro
treats the old row as `SnapshotNotFound`, replays the unchanged event log from zero, and may write a
fresh 0.2 row. Never relabel the old bytes with the new hash.

## One row, rollback-aware replacement

`keiro.keiro_snapshots` has one row per stream. For the same codec version and shape hash, a write
only replaces a row whose `streamVersion` is no newer. This prevents a late compatible write from
regressing the cache.

If either discriminant differs, a lower stream version may replace the row. That exception is what
lets a rolled-back binary reclaim the single cache slot after a newer codec wrote it. During a
mixed-version rollout, incompatible writers may thrash the row and force repeated full replays; the
event log still determines state.

This chapter reads `keiro/src/Keiro/ReadModel.hs` — the path every `runQuery` takes. Read
[01 — Snapshots in the command path](/docs/keiro/walkthrough/read-side/01-snapshots-in-the-command-path)
first.

## `runQueryWith`: validate, wait, query

```haskell
-- keiro/src/Keiro/ReadModel.hs
runQueryWith consistency readModel input = do
  schemaCheck <- ensureReadModel readModel
  case schemaCheck of
    Left err -> pure (Left err)
    Right () -> do
      waitResult <- waitIfNeeded consistency readModel
      case waitResult of
        Left err -> pure (Left err)
        Right () -> Right <$> runTransaction ((readModel ^. #query) input)
```

The order is deliberate: **schema first**, then maybe wait, then read. A read can fail before it ever
touches the table.

## `validateMetadata`: drift is a hard error

`ensureReadModel` registers the model (idempotently) and validates it against the stored row:

```haskell
validateMetadata readModel metadata
  | metadata ^. #version /= readModel ^. #version =
      stale
  | metadata ^. #shapeHash /= readModel ^. #shapeHash =
      stale
  | metadata ^. #status /= Live =
      Left (ReadModelNotLive (readModel ^. #name) (metadata ^. #status))
  | otherwise =
      Right ()
```

Because `registerReadModel` never overwrites the stored `version`/`shapeHash`, a mismatch means your
code drifted from the registered schema → `ReadModelStaleSchema`. A non-`Live` status (e.g. mid-
rebuild) → `ReadModelNotLive`. This is the read side's *hard* failure mode — the opposite of the
snapshot's silent miss — because it guards user-facing data.

## `waitIfNeeded`: where the modes diverge

```haskell
waitIfNeeded Strong _ = pure (Right ())
waitIfNeeded Eventual _ = pure (Right ())
waitIfNeeded (PositionWait options) readModel =
  case options ^. #target of
    Nothing -> pure (Right ())
    Just targetPosition -> waitFor options readModel targetPosition
```

Here is the gotcha in three lines: `Strong` and `Eventual` are the **same** — both return
immediately. A `PositionWait` with `target = Nothing` also returns immediately. Only `PositionWait`
with a `Just target` calls `waitFor`, which polls `subscriptions.last_seen` until the target
`GlobalPosition` is reached or `timeoutMicros` elapses (→ `ReadModelWaitTimeout`).

<Callout type="warn">
If you reach for `Strong` expecting it to wait for an async projection to catch up, it will not.
`Strong == Eventual` in this code path; read-your-writes against an async projection needs
`PositionWait` with the command's `GlobalPosition`. See
[Choose a consistency mode](/docs/keiro/how-to/choose-a-consistency-mode).
</Callout>

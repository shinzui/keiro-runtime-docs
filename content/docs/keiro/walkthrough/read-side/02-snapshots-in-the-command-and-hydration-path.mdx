<Callout type="warn">
  Every replay in this chapter is **aggregate replay for command hydration**. Projection replay is a
  separate event fold: it starts from projection state, consumes stored events, and never calls
  `lookupSnapshotSeed` or reads `RegFile`.
</Callout>

## `hydrate`: observe the lookup, trust full replay

```haskell
hydrate options eventStream targetStream =
  snapshotSeed >>= \case
    Nothing -> hydrateFull options eventStream targetStream
    Just seed -> do
      replayed <-
        hydrateSeeded options eventStream targetStream
          (seed ^. #state)
          (seed ^. #registers)
          (seed ^. #streamVersion)
      case replayed of
        Left _ -> hydrateFull options eventStream targetStream
        Right hydrated -> pure (Right hydrated)
  where
    snapshotSeed =
      case eventStream ^. #stateCodec of
        Nothing -> pure Nothing
        Just codec ->
          lookupSnapshotSeed resolvedName codec >>= \case
            SnapshotHit seed -> do
              recordSnapshotReadHits (options ^. #metrics) 1
              pure (Just seed)
            SnapshotUnavailable reason -> do
              recordSnapshotReadMisses (options ^. #metrics) 1
              case reason of
                SnapshotDecodeFailed _ ->
                  recordSnapshotDecodeFailures (options ^. #metrics) 1
                _ -> pure ()
              pure Nothing
```

`SnapshotNoStream`, `SnapshotNotFound`, and `SnapshotDecodeFailed` all mean "replay events from the
initial pair", but they remain distinct at the API boundary and decode failure has a dedicated
counter. A hit seeds state, registers, and version; `hydrateSeeded` reads only later events.

Even a decoded seed is advisory. If replaying its tail returns any error, keiro drops it and calls
`hydrateFull`. If full replay succeeds, the cache was wrong; if it returns the same typed replay or
gap failure, history is the problem. The event log resolves the ambiguity.

## A snapshot must cover a hidden prefix

Kiroku's per-stream truncation marker hides old events from stream reads without deleting them from
`$all`. Hydration still checks contiguous versions. A retained snapshot is sufficient only when its
version covers every hidden event—equivalently, it is at least `truncate_before - 1`. Otherwise the
first visible version is higher than expected and full replay returns
`HydrationGapDetected expected observed`.

That failure is not a snapshot decode problem. Restore visibility with
`clearStreamTruncateBefore`, or install a trusted covering snapshot as part of an operator-controlled
repair.

## `verifyAndSnapshot`: one post-commit fold

Every runner calls `verifyAndSnapshot` after a successful append or transaction:

```haskell
verifyAndSnapshot options mSpan eventStream current events appendResult
  | not (options ^. #verifyReplayOnAppend)
  , Nothing <- eventStream ^. #stateCodec = pure ()
  | otherwise =
      case Keiki.applyEventsEither transducer
             (state current, registers current) events of
        Left failure -> do
          recordSnapshotApplyDivergence (options ^. #metrics) 1
          for_ mSpan $ \sp ->
            addAttribute sp keiro_replay_divergence
              (renderReplayFailure failure)
        Right finalState ->
          case eventStream ^. #stateCodec of
            Nothing -> pure ()
            Just codec -> do
              let finalVersion = appendResult ^. #streamVersion
                  terminality = ...
              when (shouldSnapshotSpan policy terminality finalState
                      (current ^. #streamVersion) finalVersion) $ do
                encoded <- liftIO (encodeSnapshotStrict codec finalState)
                case encoded of
                  Left _ -> recordSnapshotEncodeFailures (options ^. #metrics) 1
                  Right value -> do
                    outcome <- tryError @StoreError
                      (writeSnapshotEncoded streamId finalVersion codec value)
                    case outcome of
                      Left _ -> recordSnapshotWriteFailures (options ^. #metrics) 1
                      Right () -> pure ()
```

The fold serves two purposes. With the default `verifyReplayOnAppend = True`, it witnesses a batch
that cannot replay at the exact append that poisoned the stream. With a state codec, its successful
result is also the state that may be snapshotted. Setting the option to `False` can skip the fold
only for a snapshot-less stream.

Everything in this function is post-commit. Replay divergence, strict-encode failure, and snapshot
store failure therefore cannot turn the append into `Left`: they are counted and the successful
`CommandResult` is preserved. A divergence also adds `keiro.replay.divergence` to the command span;
its compact value includes the failed event index and reason. The next hydration will report the
corresponding typed replay failure.

## Policy spans, not only final versions

The command path calls `shouldSnapshotSpan policy terminality finalState preVersion postVersion`.
For `Every n`, it fires when any positive multiple lies in `(preVersion, postVersion]`. A two-event
append from version 2 to 4 therefore satisfies `Every 3`. `OnTerminal` and `Custom` see the final pair
and terminality.

This avoids missing a policy boundary simply because one accepted command emitted several events.

## Compatibility lookup and rollback-aware upsert

`lookupSnapshot` filters before decoding:

```sql
SELECT stream_id, stream_version, state, state_codec_version,
       regfile_shape_hash, state_shape_hash, created_at, updated_at
FROM keiro.keiro_snapshots
WHERE stream_id = $1
  AND state_codec_version = $2
  AND regfile_shape_hash = $3
  AND state_shape_hash = $4
ORDER BY stream_version DESC
LIMIT 1;
```

The one-row-per-stream upsert has four ways to permit replacement:

```sql
ON CONFLICT (stream_id) DO UPDATE
  SET stream_version = EXCLUDED.stream_version,
      state = EXCLUDED.state,
      state_codec_version = EXCLUDED.state_codec_version,
      regfile_shape_hash = EXCLUDED.regfile_shape_hash,
      state_shape_hash = EXCLUDED.state_shape_hash,
      updated_at = now()
  WHERE keiro_snapshots.stream_version <= EXCLUDED.stream_version
     OR keiro_snapshots.state_codec_version <> EXCLUDED.state_codec_version
     OR keiro_snapshots.regfile_shape_hash <> EXCLUDED.regfile_shape_hash
     OR keiro_snapshots.state_shape_hash <> EXCLUDED.state_shape_hash;
```

The first clause prevents version regression only for a fully compatible row. **Any** discriminant
changing permits a lower version to replace a higher one, which makes codec rollback possible.
Incompatible binaries can therefore thrash the cache row during a mixed deployment; they cannot
change event history.

`state_shape_hash` is the third column, added by
`0019-keiro-snapshots-state-shape-hash.sql` with an empty-string default — so every pre-migration row
misses exactly once and is re-persisted with a real hash.

## The sampled seed witness

An accepted seed is only as trustworthy as the discriminator that accepted it, and the discriminator
cannot see a hand-written or Holes-only fold edit. So `hydrate` samples: on a snapshot hit, one
hydration in `RunCommandOptions.seedVerifySampleRate` (default 1000) spawns a background replay
bounded **through the seed's own stream version** and compares RFC 8785 canonical encoded state
against the seed.

```haskell
Right hydrated -> do
  for_ (eventStream ^. #stateCodec) $ \codec ->
    scheduleSeedVerification options eventStream targetStream codec seed
  pure (Right hydrated)
```

It never blocks or fails the command and never writes a snapshot. A mismatch increments
`keiro.snapshot.seed.divergence` and writes a structured stderr line with the stream, seed version,
and both SHA-256 digests. Set the rate to `1` to verify every hit, `0` to disable it.

## Read the signals together

- Rising `keiro.snapshot.read.misses` after a planned codec or keiki 0.2 upgrade is expected while
  rows repopulate. Sustained misses suggest mixed versions or a policy that never fires.
- `keiro.snapshot.decode.failures` means the compatibility keys matched but bytes did not. Full
  replay protects correctness; inspect or evict the row.
- `keiro.snapshot.encode.failures` and `keiro.snapshot.write.failures` mean the events committed but
  the cache did not advance.
- `keiro.snapshot.apply.divergence` is more serious: the events committed and the aggregate can no
  longer replay them. Stop further commands for the affected stream and investigate immediately.

The [Snapshot reference](/docs/keiro/reference/snapshot#caller-visible-and-operational-outcomes)
puts these cases in one committed-state/result/fallback table.

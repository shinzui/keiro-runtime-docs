## The `ReadModel` and its consistency modes

A read model is a queryable projection table plus the metadata that guards it:

```haskell
-- keiro/src/Keiro/ReadModel.hs
data ReadModel q r = ReadModel
  { name :: !Text
  , tableName :: !Text
  , subscriptionName :: !Text
  , version :: !Int
  , shapeHash :: !Text
  , defaultConsistency :: !ConsistencyMode
  , query :: !(q -> Tx.Transaction r)
  }

data ConsistencyMode
  = Strong
  | Eventual
  | PositionWait !PositionWaitOptions
  deriving stock (Generic, Eq, Show)

data PositionWaitOptions = PositionWaitOptions
  { target :: !(Maybe GlobalPosition)   -- Nothing => skip waiting
  , timeoutMicros :: !Int
  , pollMicros :: !Int
  }
```

The errors a query can return:

```haskell
-- keiro/src/Keiro/ReadModel.hs
data ReadModelError
  = ReadModelStaleSchema !Text !Int !Int !Text !Text
  | ReadModelWaitTimeout !Text !GlobalPosition !GlobalPosition
  | ReadModelNotLive     !Text !ReadModelStatus
  deriving stock (Generic, Eq, Show)
```

## `runQueryWith`: validate, wait, query

```haskell
-- keiro/src/Keiro/ReadModel.hs
runQueryWith consistency readModel input = do
  schemaCheck <- ensureReadModel readModel
  case schemaCheck of
    Left err -> pure (Left err)
    Right () -> do
      waitResult <- waitIfNeeded consistency readModel
      case waitResult of
        Left err -> pure (Left err)
        Right () -> Right <$> runTransaction ((readModel ^. #query) input)
```

The order is deliberate: **schema first**, then maybe wait, then read. A read can fail before it ever
touches the table. (`runQuery` is just `runQueryWith` applied to the model's `defaultConsistency`.)

## `validateMetadata`: drift is a hard error

`ensureReadModel` registers the model (idempotently) and validates it against the stored registry row
via `validateMetadata`:

```haskell
-- keiro/src/Keiro/ReadModel.hs
validateMetadata readModel metadata
  | metadata ^. #version /= readModel ^. #version = stale
  | metadata ^. #shapeHash /= readModel ^. #shapeHash = stale
  | metadata ^. #status /= Live = Left (ReadModelNotLive (readModel ^. #name) (metadata ^. #status))
  | otherwise = Right ()
```

Because `registerReadModel` (below) **never overwrites** the stored `version`/`shapeHash`, a mismatch
means your code drifted from the registered schema → `ReadModelStaleSchema`. A non-`Live` status (for
example mid-rebuild) → `ReadModelNotLive`.

<Callout type="info">
**Note the failure-mode asymmetry with snapshots.** Schema drift here is a *hard* failure that aborts
the read — it guards user-facing data. A snapshot miss (chapter 02) is the opposite: it is *silent*
and just falls back to a full replay. Read models fail loud; snapshots fail quiet.
</Callout>

## `waitIfNeeded`: where the modes diverge

```haskell
-- keiro/src/Keiro/ReadModel.hs
waitIfNeeded Strong _ = pure (Right ())
waitIfNeeded Eventual _ = pure (Right ())
waitIfNeeded (PositionWait options) readModel =
  case options ^. #target of
    Nothing -> pure (Right ())
    Just targetPosition -> waitFor options readModel targetPosition
```

Here is the gotcha in three lines: `Strong` and `Eventual` are the **same** — both return `Right ()`
immediately. A `PositionWait` with `target = Nothing` also returns immediately. Only `PositionWait`
with a `Just target` calls `waitFor`, which polls `subscriptions.last_seen` until the target
`GlobalPosition` is reached or `timeoutMicros` elapses (→ `ReadModelWaitTimeout`). `waitFor`'s poll
SQL:

```sql
SELECT last_seen
FROM subscriptions
WHERE subscription_name = $1
```

(The `subscriptions` table is **kiroku-owned** — it is not in keiro's own migration.)

<Callout type="warn">
If you reach for `Strong` expecting it to wait for an async projection to catch up, it will not.
`Strong == Eventual` in this code path; both skip the wait. Read-your-writes against an *async*
projection needs `PositionWait` with the command's `GlobalPosition` as `target`. (An *inline*
projection, by contrast, already committed with the events — see chapter 04.) See
[Choose a consistency mode](/docs/keiro/how-to/choose-a-consistency-mode).
</Callout>

## The `keiro_read_models` registry

The registry lives in `Keiro.ReadModel.Schema`:

```haskell
-- keiro/src/Keiro/ReadModel/Schema.hs
data ReadModelStatus = Live | Rebuilding | Paused | Abandoned
  deriving stock (Generic, Eq, Show)

data ReadModelMetadata = ReadModelMetadata
  { name :: !Text
  , version :: !Int
  , shapeHash :: !Text
  , lastBuiltAt :: !(Maybe UTCTime)
  , status :: !ReadModelStatus
  }

registerReadModel :: (Store :> es) => Text -> Int -> Text -> Eff es ReadModelMetadata
lookupReadModel   :: (Store :> es) => Text -> Eff es (Maybe ReadModelMetadata)
markRebuilding    :: (Store :> es) => Text -> Int -> Text -> Eff es ReadModelMetadata
markLive          :: (Store :> es) => Text -> Int -> Text -> Eff es ReadModelMetadata
markAbandoned     :: (Store :> es) => Text -> Int -> Text -> Eff es ReadModelMetadata
```

`registerReadModel`'s statement is an **idempotent** insert that returns the *existing* row unchanged
when one is already present — it never overwrites the stored `version`/`shape_hash`:

```sql
-- registerReadModelStmt: insert-if-absent, then return whichever row stands
INSERT INTO keiro_read_models (name, version, shape_hash, status, last_built_at)
VALUES ($1, $2, $3, 'live', now())
ON CONFLICT (name) DO NOTHING
RETURNING name, version, shape_hash, last_built_at, status
-- … UNION ALL with a SELECT of the existing row, so the caller always gets the stored values
```

That "never overwrite" behavior is exactly what lets `validateMetadata` detect drift: the stored row
is the source of truth, and your in-code `version`/`shapeHash` are compared against it. The three
status transitions (`markRebuilding`/`markLive`/`markAbandoned`) all go through one
`transitionReadModelStmt` upsert. Decoding is defensive: `statusFromText` maps an unrecognized status
text to `Paused` rather than failing the read.

The DDL:

```sql
-- keiro-migrations/.../2026-05-17-00-00-00-keiro-bootstrap.sql
CREATE TABLE IF NOT EXISTS keiro_read_models (
  name          TEXT PRIMARY KEY,
  version       BIGINT NOT NULL,
  shape_hash    TEXT NOT NULL,
  last_built_at TIMESTAMPTZ,
  status        TEXT NOT NULL,           -- no CHECK constraint on status
  updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

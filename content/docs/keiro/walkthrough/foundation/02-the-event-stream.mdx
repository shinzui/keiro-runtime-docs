## What this part covers

`keiro-core/src/Keiro/EventStream.hs` and `keiro-core/src/Keiro/EventStream/Validate.hs`. The first
module defines the raw record that, in the module's own words, "marries a pure keiki
`SymTransducer` … with everything keiro needs to run it against a durable event store." The second
module defines the validation wrapper that public runners now require. We read the seven raw fields,
the two helper types it carries (`SnapshotPolicy` and `StateCodec`), and then the
`ValidatedEventStream` boundary that keeps unchecked streams out of the write path.

## The marriage, field by field

```haskell
-- keiro-core/src/Keiro/EventStream.hs
data EventStream phi rs s ci co = EventStream
  { transducer :: !(SymTransducer phi rs s ci co)
  , initialState :: !s
  , initialRegisters :: !(RegFile rs)
  , eventCodec :: !(Codec co)
  , resolveStreamName :: !(Stream (EventStream phi rs s ci co) -> StreamName)
  , snapshotPolicy :: !(SnapshotPolicy (s, RegFile rs))
  , stateCodec :: !(Maybe (StateCodec (s, RegFile rs)))
  }
  deriving stock (Generic)
```

And the seven fields:

<TypeTable
  type={{
    transducer: { description: "The pure keiki state machine that turns a command into emitted events. The decision core.", type: "SymTransducer phi rs s ci co" },
    initialState: { description: "The machine's starting control state, used when hydrating an empty stream.", type: "s" },
    initialRegisters: { description: "The machine's starting register file, used when hydrating an empty stream.", type: "RegFile rs" },

    snapshotPolicy: { description: "Decides, per append, whether to persist a snapshot of the (state, registers) pair.", type: "SnapshotPolicy (s, RegFile rs)" },
    stateCodec: { description: "How to serialize that snapshot. Nothing disables snapshotting regardless of snapshotPolicy.", type: "Maybe (StateCodec (s, RegFile rs))" },
  }}
/>

The first three fields *seed* a fresh machine; `eventCodec` is the [serialization
boundary](/docs/keiro/walkthrough/foundation/03-the-codec); `resolveStreamName` turns the
[typed handle](/docs/keiro/walkthrough/foundation/01-the-stream-handle) into a physical name; and the
last two govern snapshotting.

Look closely at `snapshotPolicy` and `stateCodec`. Both are parameterized over **`(s, RegFile rs)`** —
the *pair* of folded control state and register file, not just `s`. That is the structural proof,
visible right in the type, that a snapshot concerns **both** halves of the machine: where it sits *and*
its working memory. The next two chapters explain why the registers are a separate bank that cannot be
recovered from the state alone.

## SnapshotPolicy — when to snapshot

```haskell
-- keiro-core/src/Keiro/EventStream.hs
data SnapshotPolicy state
  = Never
  | Every !Int
  | OnTerminal
  | Custom !(Terminality -> state -> StreamVersion -> Bool)
  deriving stock (Generic)
```

<TypeTable
  type={{
    Never: { description: "Never snapshot; always rehydrate from the full event log.", type: "constructor" },
    Every: { description: "Snapshot whenever the stream version is a multiple of n. A non-positive interval disables snapshotting.", type: "!Int" },
    OnTerminal: { description: "Snapshot only when the machine has reached a final state.", type: "constructor" },
    Custom: { description: "An arbitrary predicate over the folded state and the current StreamVersion.", type: "!(state -> StreamVersion -> Bool)" },
  }}
/>

## StateCodec — how to serialize a snapshot

```haskell
-- keiro-core/src/Keiro/EventStream.hs
data StateCodec state = StateCodec
  { stateCodecVersion :: !Int
  , shapeHash :: !Text
  , encode :: !(state -> Value)
  , decode :: !(Value -> Either Text state)
  }
  deriving stock (Generic)
```

<TypeTable
  type={{
    stateCodecVersion: { description: "Bumped when the snapshot encoding changes incompatibly.", type: "!Int" },
    shapeHash: { description: "A digest of the folded-state shape; protects against silently loading a snapshot whose structure no longer matches.", type: "!Text" },
    encode: { description: "JSON serialization of the (state, registers) pair.", type: "!(state -> Value)" },
    decode: { description: "JSON deserialization of the (state, registers) pair.", type: "!(Value -> Either Text state)" },
  }}
/>

The Haddock states the load-bearing rule: `stateCodecVersion` and `shapeHash` **together** gate
snapshot reuse. A stored snapshot is loaded only when *both* match the current codec, so changing the
snapshot encoding or the shape of the folded state invalidates older snapshots and forces a clean
rehydration from events. That is what protects you from silently decoding a snapshot whose structure no
longer matches the code.

## The validated wrapper

The raw record above is only the definition. The runtime boundary is a hidden-constructor wrapper:

```haskell
-- keiro-core/src/Keiro/EventStream/Validate.hs
newtype ValidatedEventStream phi rs s ci co =
  ValidatedEventStream (EventStream phi rs s ci co)

unvalidated :: ValidatedEventStream phi rs s ci co -> EventStream phi rs s ci co
```

Application code cannot construct `ValidatedEventStream` directly. It must pass the raw record
through one of the smart constructors:

```haskell
mkEventStream ::
  (Bounded s, Enum s, Ord s, Show s) =>
  Text ->
  EventStream (HsPred rs ci) rs s ci co ->
  Either [EventStreamWarning] (ValidatedEventStream (HsPred rs ci) rs s ci co)

mkEventStreamWith ::
  (Bounded s, Enum s, Ord s, Show s) =>
  ValidationOptions ->
  Text ->
  EventStream (HsPred rs ci) rs s ci co ->
  Either [EventStreamWarning] (ValidatedEventStream (HsPred rs ci) rs s ci co)

mkEventStreamOrThrow ::
  (HasCallStack, Bounded s, Enum s, Ord s, Show s) =>
  Text ->
  EventStream (HsPred rs ci) rs s ci co ->
  ValidatedEventStream (HsPred rs ci) rs s ci co

mkEventStreamUnchecked ::
  EventStream phi rs s ci co ->
  ValidatedEventStream phi rs s ci co
```

`mkEventStream` runs two classes of checks:

- **keiki transducer validation** via `validateTransducer`: hidden input, first-event
  recoverability, inversion ambiguity, guarded input reads, state-changing output-free edges,
  structural determinism, and structural dead-edge detection under `defaultValidationOptions`.
- **keiro stream-level validation**: any policy other than `Never` with `stateCodec = Nothing` is
  rejected, and a present codec is forced over the initial `(state, registers)` so an uninitialized
  register thunk fails before the service accepts commands.

Keiro is stricter than a generic keiki caller. `mkEventStreamWith` applies custom options but then
force-enables `checkHeadRecoverability` and `checkStateChangingEpsilon`; persisted events are the
only durable state, so those checks cannot be opted out of at this boundary. Every remaining enabled
warning also rejects construction.

On success the smart constructor returns `Right validated`. On failure it returns labelled
`EventStreamWarning`s whose `eswStreamLabel` is the caller's label and whose `eswReason` renders the
specific hidden-input, head-recoverability, inversion-ambiguity, guarded-read, state-changing-
epsilon, nondeterminism, dead-edge, opaque-guard, or snapshot problem. The
public command runners unwrap with `unvalidated` internally, but only after the type system has forced
callers to present this validated wrapper.

`mkEventStreamOrThrow` is partial and belongs in generated modules or fixtures whose sibling tests
prove validation remains green. `mkEventStreamUnchecked` skips every check and belongs only in tests
or emergency forensics; using it in application wiring removes the durable boundary entirely.

## The load-bearing facts

1. **A raw `EventStream` is no longer runnable.** It is the construction shape, not the command-runner
   input. `runCommand`, `runCommandWithSql`, projections, routers, and process managers require
   `ValidatedEventStream`.
2. **`stateCodec = Nothing` disables snapshotting — regardless of `snapshotPolicy`.** The type makes
   the dependency explicit: `stateCodec` is a `Maybe`. With no codec there is no way to serialize a
   snapshot, so even an aggressive `snapshotPolicy` like `Every 1` writes nothing. `mkEventStream`
   now rejects that incoherent pair instead of letting it become a silent no-op.
3. **Snapshots are about the pair.** Because `snapshotPolicy` and `stateCodec` are both over
   `(s, RegFile rs)`, the persisted blob always carries the control state *and* the registers.
4. **A no-op cannot hide a durable change.** Keiro always rejects an output-free edge that changes
   the control vertex or can write a register. Without an event, hydration could not recover it.

## The jitsurei anchor — both ends

The order example shows both configurations side by side
(`jitsurei/src/Jitsurei/OrderStream.hs`). First, the no-snapshot marriage:

```haskell
-- jitsurei/src/Jitsurei/OrderStream.hs
orderEventStreamDef :: OrderEventStream
orderEventStreamDef = EventStream
  { transducer = orderTransducer
  , initialState = NotStarted
  , initialRegisters = RNil               -- OrderRegs = '[]: an empty register file
  , eventCodec = orderCodec
  , resolveStreamName = Stream.streamName
  , snapshotPolicy = Never
  , stateCodec = Nothing
  }

orderEventStream :: ValidatedOrderEventStream
orderEventStream =
  mkEventStreamOrThrow "jitsurei-order" orderEventStreamDef
```

where `type OrderEventStream = EventStream (HsPred OrderRegs OrderCommand) OrderRegs OrderState
OrderCommand OrderEvent`, `type ValidatedOrderEventStream = ValidatedEventStream (HsPred OrderRegs
OrderCommand) OrderRegs OrderState OrderCommand OrderEvent`, and `type OrderRegs = '[]`. The order
aggregate carries *no* registers (`RNil`), so it is the cleanest place to see the wiring without
register noise. It also opts out of snapshotting on both counts: `snapshotPolicy = Never` and
`stateCodec = Nothing`. The runnable value is the validated wrapper, not `orderEventStreamDef`.

Then the same stream with snapshotting switched on — note it overrides *both* fields:

```haskell
-- jitsurei/src/Jitsurei/OrderStream.hs
snapshotOrderEventStreamDef :: OrderEventStream
snapshotOrderEventStreamDef =
  orderEventStreamDef
    { snapshotPolicy = Every 2
    , stateCodec = Just (defaultStateCodec @OrderRegs @OrderState 1)
    }

snapshotOrderEventStream :: ValidatedOrderEventStream
snapshotOrderEventStream =
  mkEventStreamOrThrow "jitsurei-order-snapshot" snapshotOrderEventStreamDef
```

`Every 2` fires the policy on even versions, and `Just (defaultStateCodec @OrderRegs @OrderState 1)`
supplies the codec. Setting one without the other would be a silent no-op — which is exactly why both
appear together.

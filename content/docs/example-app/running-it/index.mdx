---
title: "Running it"
description: "Clone, build, migrate, and run keiro-runtime-jitsurei's three packaged scenarios, then inspect the traces in Jaeger."
---

`keiro-runtime-jitsurei` is meant to be run. This subsection takes you from a clean checkout to a
cross-service trace in Jaeger: the Nix dev shell and local infrastructure
(`process-compose.yaml`), the database and runtime migrations, the three packaged scenarios
(`mass-casualty-transfer`, `hospital-divert-reroute`, `supply-shortage-escalation`), the
per-service workers, and how to read the resulting traces.

<Callout type="warn">
  Scenario `--reset` **drops and recreates the two local development databases** (and recreates the
  Kafka topics). Never point it at a non-development connection string.
</Callout>

<Callout type="info">
  The chapters of this tour land with [Milestone 5](/docs/example-app/overview/03-runtime-feature-map)
  of the documentation plan. Start with the [Overview](/docs/example-app/overview) for the domain
  and the runtime-feature map.
</Callout>

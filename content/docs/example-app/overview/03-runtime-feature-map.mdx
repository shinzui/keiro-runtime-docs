This is the index that ties the whole section together. For every keiro runtime feature it names
the **exact app file/function**, the **chapter in this section** that walks it, and the
**canonical keiro reference page**. Later chapters link back here; treat it as the map you keep open
in another tab.

The "Covered in" column links to the relevant service/section tour and names the specific chapter;
once you are inside a tour, its `meta.json` ordering takes you chapter to chapter.

---
title: shibuya
description: A supervised, Broadway-style queue-processing framework with backpressure and batching.
---

shibuya (渋谷, the Tokyo district) is a supervised queue-processing framework — Broadway-style concurrent
message and job pipelines with backpressure, batching, and rate limiting. It
reliably consumes message queues on behalf of the keiro runtime.

shibuya reads each backend through an **adapter**: the
[shibuya ⇄ kiroku adapter](/docs/integrations/shibuya-kiroku-adapter) pulls from the
kiroku event store, and the [shibuya ⇄ pgmq adapter](/docs/integrations/shibuya-pgmq-adapter)
consumes a [pgmq](/docs/pgmq) message queue.

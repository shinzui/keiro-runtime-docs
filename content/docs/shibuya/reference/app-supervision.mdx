## Supervision strategy

```haskell
data SupervisionStrategy
  = IgnoreFailures
  | StopAllOnFailure
```

`IgnoreFailures` keeps other processors running if one child exits. `StopAllOnFailure` maps to an
NQE strategy that stops all processors when one fails. Graceful exits, including finite streams
completing and handlers returning `AckHalt`, do not stop sibling processors.

## Application config

```haskell
data AppConfig = AppConfig
  { strategy :: SupervisionStrategy
  , inboxSize :: Int
  }

defaultAppConfig :: AppConfig
```

`defaultAppConfig` is `IgnoreFailures` with an inbox of `100`. `runApp` validates that `inboxSize`
is at least `1` before starting any processor.

## Shutdown configuration

```haskell
data ShutdownConfig = ShutdownConfig
  { drainTimeout :: NominalDiffTime
  }

defaultShutdownConfig :: ShutdownConfig
```

The default drain timeout is 30 seconds.

## App handle operations

```haskell
getAppMetrics :: (IOE :> es) => AppHandle es -> Eff es MetricsMap

stopApp :: (IOE :> es) => AppHandle es -> Eff es ()

stopAppGracefully ::
  (IOE :> es) =>
  ShutdownConfig ->
  AppHandle es ->
  Eff es Bool

waitApp :: (IOE :> es) => AppHandle es -> Eff es ()
```

`stopApp` uses `defaultShutdownConfig`. `stopAppGracefully` first calls every adapter's `shutdown`,
then waits for in-flight messages to drain, then stops the master. It returns `True` when all
processors drained before timeout.

## Structured errors

```haskell
data AppError
  = AppPolicyError PolicyError
  | AppHandlerError HandlerError
  | AppRuntimeError RuntimeError
  | AppBatchConfigError BatchConfigError
  | AppConfigInvalid ConfigError
```

Policy, batch config, and app config errors are produced before processors start. Runtime errors
cover supervisor failures. Handler exceptions are represented as `HandlerException Text`; on the
single-message path, thrown handlers are finalized as `AckRetry (RetryDelay 0)`.

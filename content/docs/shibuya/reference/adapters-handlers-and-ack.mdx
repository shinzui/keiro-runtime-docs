## Adapter

```haskell
data Adapter es msg = Adapter
  { adapterName :: Text
  , source :: Stream (Eff es) (Ingested es msg)
  , shutdown :: Eff es ()
  }
```

`source` yields leased messages. `shutdown` stops polling and releases backend resources during
graceful stop. `adapterName` is for logging and observability.

## Handler

```haskell
type Handler es msg = Message es msg -> Eff es AckDecision
```

The handler returns intent only. It receives `Message`, not `Ingested`, so application code can read
the envelope and optional lease but cannot call the adapter finalizer directly. The framework owns
finalization and applies the returned decision to the original `AckHandle`. If the handler throws,
the single-message runner finalizes the delivery with `AckRetry (RetryDelay 0)` and uses the same
bounded finalizer retry path as batch processing.

An adapter finalizer must be idempotent for the same decision. The runner never intentionally
finalizes one delivery with two different decisions, but it may retry the same `finalize decision`
after a transient finalizer failure.

## Ack decisions

```haskell
newtype RetryDelay = RetryDelay { unRetryDelay :: NominalDiffTime }

data DeadLetterReason
  = PoisonPill Text
  | InvalidPayload Text
  | MaxRetriesExceeded

data HaltReason
  = HaltOrderedStream Text
  | HaltFatal Text

data AckDecision
  = AckOk
  | AckRetry RetryDelay
  | AckDeadLetter DeadLetterReason
  | AckHalt HaltReason
```

`AckRetry` carries an absolute delay duration, not a retry count. `AckDeadLetter` records why this
message should leave normal processing. `AckHalt` stops processor execution.

## Mock adapter helpers

```haskell
listAdapter :: (IOE :> es) => [Ingested es msg] -> Adapter es msg

data TrackingAck = TrackingAck

newTrackingAck :: (IOE :> es) => Eff es TrackingAck

trackingAckHandle ::
  (IOE :> es) =>
  TrackingAck ->
  MessageId ->
  AckHandle es

getTrackedDecisions ::
  (IOE :> es) =>
  TrackingAck ->
  Eff es [(MessageId, AckDecision)]

mkTrackedIngested :: (IOE :> es) => TrackingAck -> Envelope msg -> Ingested es msg

trackedListAdapter :: (IOE :> es) => TrackingAck -> [Envelope msg] -> Adapter es msg
```

Use these helpers to test handlers and runner behavior without a real queue backend.

## First-party adapters

The core package defines the contract, while adapter packages bind it to concrete
backends. See [shibuya adapter comparison](/docs/integrations/shibuya-adapters)
for source-checked notes on the pgmq, Kiroku, Kafka, and Message DB adapters.

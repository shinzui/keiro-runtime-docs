---
title: "Alert on failures"
description: "Prometheus alert rules for failed processors and message failures."
---

Alert when any processor is failed:

```yaml
- alert: ShibuyaProcessorFailed
  expr: shibuya_processor_state == 3
  for: 1m
  labels:
    severity: page
  annotations:
    summary: "shibuya processor {{ $labels.processor }} is failed"
```

Alert on sustained message failures:

```yaml
- alert: ShibuyaMessageFailures
  expr: rate(shibuya_messages_failed_total[5m]) > 0
  for: 5m
  labels:
    severity: ticket
  annotations:
    summary: "shibuya processor {{ $labels.processor }} is failing messages"
```

Use `/health/ready` for stuck-processor detection. Current Prometheus output does not expose a
last-activity timestamp.

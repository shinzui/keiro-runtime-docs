---
title: "Add a health endpoint"
description: "Expose liveness and readiness probes from the shibuya metrics server."
---

```haskell
withMetricsServer defaultConfig (getAppMaster appHandle) $ \_server ->
  runEff (runTracingNoop (waitApp appHandle))
```

Probe it:

```bash
curl -f http://127.0.0.1:9090/health/live
curl -f http://127.0.0.1:9090/health/ready
```

Use `/health/live` for process liveness and `/health/ready` for traffic readiness. Readiness fails
when dependencies fail, a processor fails, or a processor is stuck beyond `stuckThreshold`.

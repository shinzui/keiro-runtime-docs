---
title: "Test with the mock adapter"
description: "Use listAdapter and trackingAckHandle to run handlers against finite messages."
---

`Shibuya.Adapter.Mock` is the test adapter in `shibuya-core`. It provides finite list-backed
adapters and ack handles that record every decision.

## Create tracked messages

The helper `newTrackingAck` allocates an in-memory decision log. `trackedListAdapter` wraps
envelopes in `Ingested` values whose ack handles append `(MessageId, AckDecision)` to that log.

```haskell
mkTestEnvelope :: MessageId -> msg -> Envelope msg
mkTestEnvelope msgId payload =
  (mkEnvelope msgId payload)
    { attempt = Just (Attempt 0)
    }
```

## Feed a finite adapter

`trackedListAdapter` wraps a list of envelopes in an `Adapter` whose stream ends after the list is
drained.

```haskell
runOneBatch :: Eff '[Tracing, IOE] [(MessageId, AckDecision)]
runOneBatch = do
  tracking <- newTrackingAck
  let env1 = mkTestEnvelope (MessageId "one") (OrderJob "o-1" 1200)
      env2 = mkTestEnvelope (MessageId "two") (OrderJob "o-2" 0)
      adapter = trackedListAdapter tracking [env1, env2]

  result <-
    runApp
      defaultAppConfig {inboxSize = 10}
      [(ProcessorId "orders-test", mkProcessor adapter handleOrder)]

  case result of
    Left err -> liftIO (fail (show err))
    Right appHandle -> waitApp appHandle

  getTrackedDecisions tracking
```

`waitApp` returns after the finite adapter stream is exhausted and the inbox is drained.

## Assert decisions, not backend effects

The mock adapter verifies your handler contract at the shibuya boundary. Assert that the expected
message id received the expected `AckDecision`; leave broker-specific effects to adapter tests.

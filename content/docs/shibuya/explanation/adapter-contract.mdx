---
title: "The adapter contract"
description: "The source, ack, lease, and metadata responsibilities every shibuya backend adapter owns."
---

An adapter is the only code that speaks both shibuya and a backend queue or event source.

```haskell
data Adapter es msg = Adapter
  { adapterName :: Text
  , source :: Stream (Eff es) (Ingested es msg)
  , shutdown :: Eff es ()
  }
```

The adapter owns polling, decoding, lease acquisition, cursor or offset tracking, and the backend
effect of each `AckDecision`. The runner owns none of that. It only consumes `source`, invokes
handlers, and calls the `AckHandle` inside each `Ingested`.

That split is why adapters can vary widely. A pgmq adapter can derive `Attempt` from the queue row's
read count and implement retry with visibility timeouts. A Kafka adapter can set partition metadata
and commit offsets. A kiroku adapter can expose event-store positions as cursors. A mock adapter can
record decisions in an `IORef`.

Adapters should fill the `Envelope` as faithfully as the backend allows. Use `Nothing` when the
backend cannot surface a field. For headers, `Nothing` means the adapter does not surface headers at
all, while `Just []` means the adapter surfaces headers and this message had none.

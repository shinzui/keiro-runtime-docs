title: Write-Path Walkthrough
description: An ordered tour through kiroku's real append source — the Store effect, the single-statement append CTE, optimistic concurrency, and transactional appends.
---

An ordered tour that reads kiroku's **real append source** to teach how a write actually lands in the
event store. It is the counterpart to the [Subscription Walkthrough](/docs/kiroku/walkthrough) and is
**completely independent of it** — nothing here touches subscriptions, the worker, or the publisher.
The write path has one organising idea, and we follow it the whole way: **every append is a single
SQL statement** that is atomic and enforces optimistic concurrency _in the database_, not in Haskell.

<Callout type="info">
  Read the parts in order, starting at [00 — Start here](/docs/kiroku/write-path/00-start-here). It
  assumes you have met the concepts from the user's point of view in the explanation pages
  ([Append-only log](/docs/kiroku/explanation/append-only-log), [Optimistic
  concurrency](/docs/kiroku/explanation/optimistic-concurrency), [The $all stream and global
  order](/docs/kiroku/explanation/all-stream-and-global-order)); here we go under the hood and read
  the code that implements them.
</Callout>

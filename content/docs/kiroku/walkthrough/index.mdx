An ordered tour that reads kiroku's real source code to teach how the **subscription** side is built.
It follows the subscription machinery end to end — the pure state machine, the impure worker driver,
and the lifecycle handle — then crosses into the **shibuya adapter** that bridges that machinery into
shibuya's queue-processing runtime. These are the parts with the most moving pieces and the
clearest payoff from reading the source. For the append side, see the separate [Write-Path
Walkthrough](/docs/kiroku/write-path).

</Callout>

Chapter 07 read the runtime codec that works against a type-level slot list. Most callers never write a
`RegFile` by hand, though — they have an ordinary record or an event sum. Two Template Haskell splices
in `keiki-codec-json` close that gap by emitting the codec functions for you: `deriveRegFileCodec` for
a single-record snapshot, and `deriveEventCodecSkeleton` for an event sum. Both live in
`keiki-codec-json` rather than `keiki` core precisely because the generated code references `aeson`,
and keiki core must stay aeson-free.

## `deriveRegFileCodec`: the record snapshot

For a record type, `$(deriveRegFileCodec ''MySnapshot)` emits three top-level functions, with names
derived by lower-casing the first character of the type name:

```haskell
-- keiki-codec-json/src/Keiki/Codec/JSON/TH.hs (Haddock)
mySnapshotToJSON     :: MySnapshot -> Aeson.Value
mySnapshotToEncoding :: MySnapshot -> Aeson.Encoding
mySnapshotFromJSON   :: Aeson.Value -> Either String MySnapshot
```

Each one routes through the `RegFileToJSON` class from chapter 07 against the slot list
`RegFieldsOf MySnapshot`. The record's field names become the JSON object keys; each field type must
carry `ToJSON` and `FromJSON` or compilation fails with a per-field error. The splice body is just three
function definitions that compose the generic bridge with the runtime codec:

```haskell
-- keiki-codec-json/src/Keiki/Codec/JSON/TH.hs
  toJSONDef <- funD toJSONN
                 [ clause [] (normalB
                     [| regFileToJSON
                          @(RegFieldsOf $recTy)
                          . gToRegFile . Generics.from |])
                     []
                 ]
```

`Generics.from` turns the record into its generic representation, `gToRegFile` projects that into a
`RegFile`, and `regFileToJSON` (chapter 07) encodes it. The record only needs `deriving (Generic)`; the
splice does not emit a `Generic` instance.

### What the splice rejects — and what it accepts

`validateRecord` and `validateCtors` reject every shape that cannot map onto a single slot list, each
with a precise message:

```haskell
-- keiki-codec-json/src/Keiki/Codec/JSON/TH.hs
validateCtors tyName ctors = case ctors of
  [RecC _ _]     -> pure ()
  [NormalC _ []] -> pure ()
  [NormalC _ _]  -> fail $ "deriveRegFileCodec: " <> show tyName
                        <> " has a positional (non-record-syntax) "
                        <> "constructor; switch to record syntax so "
                        <> "field names are available as slot symbols, "
                        <> "e.g. `data Foo = Foo { x :: Int }`."
  []             -> fail $ "deriveRegFileCodec: " <> show tyName
                        <> " has no constructors; pass a record type."
  (_ : _ : _)    -> fail $ "deriveRegFileCodec: " <> show tyName
                        <> " has multiple constructors; a single slot "
                        <> "list cannot represent a sum type."
  _              -> fail $ ...
```

<Cards>
  <Card title="Rejected: type synonyms" description="Only data and newtype declarations are accepted; a TySynD fails." />
  <Card title="Rejected: sum types" description="Multi-constructor data Foo = A | B — a single slot list cannot represent a sum." />
  <Card title="Rejected: positional ctors" description="No field names means no slot symbols; switch to record syntax." />
  <Card title="Accepted: singleton" description="data Empty = Empty gives the empty slot list '[]; the functions codec the empty JSON object {}." />
</Cards>

The singleton no-field constructor is the deliberate edge case: its slot list is `'[]`, and the emitted
functions encode it to — and decode it from — the empty JSON object. `THSpec` pins this with an `Empty`
type, asserting the streaming path emits the literal bytes `{}` and that a non-empty object is rejected
as unknown extra fields.

<Callout type="info">
**Test anchor.** `Keiki.Codec.JSON.THSpec` drives `deriveRegFileCodec` against a two-field `TestRec`
and the singleton `Empty`: round-trip, encoding agreement, slot-list-order key emission, and the three
strict-decoder rejections (missing field, unknown extra field, type-mismatched field — each
slot-prefixed). It documents the compile-fail negative cases as a manual procedure, since a compile
failure cannot be a passing unit test.
</Callout>

## `deriveEventCodecSkeleton`: the event sum

Where the record splice handles one product, this one handles a *sum* — a `data` type with several
constructors, each wrapping a single record payload (or a no-arg singleton). For

```haskell
-- keiki-codec-json/src/Keiki/Codec/JSON/Event.hs (Haddock)
data OrderEvent
  = Placed PlacedData
  | Shipped ShippedData
  | Cancelled
  deriving stock (Eq, Show)
```

the splice `$(deriveEventCodecSkeleton opts ''OrderEvent)` emits **five** top-level bindings:

```haskell
-- keiki-codec-json/src/Keiki/Codec/JSON/Event.hs (Haddock)
orderEventToJSON     :: OrderEvent -> Aeson.Value
orderEventFromJSON   :: Aeson.Value -> Either String OrderEvent
orderEventEventTypes :: [Data.Text.Text]                    -- wire kinds, in order
orderEventKindMap    :: [(Data.Text.Text, Data.Text.Text)]  -- (ctor, wire kind)
orderEventSchemaVersion :: Int
```

<TypeTable
  type={{
    "<p>ToJSON": { type: "Event -> Aeson.Value", description: "One encode clause per constructor; each emits a kind discriminator plus one entry per payload field." },
    "<p>FromJSON": { type: "Aeson.Value -> Either String Event", description: "Reads kind, branches on it, and reassembles the payload field by field." },
    "<p>EventTypes": { type: "[Text]", description: "Resolved wire kinds in declaration order — a Keiro-feeding surface, plain Text, no Keiro import." },
    "<p>KindMap": { type: "[(Text, Text)]", description: "Each (constructor name, resolved wire kind) pair." },
    "<p>SchemaVersion": { type: "Int", description: "The version stamped into every envelope by this generated codec." },
  }}
/>

### The `kind` discriminator

Each constructor encodes to a JSON object carrying a resolved `"kind"`, an in-band `"v"`, and one
entry per payload field. The encoder generates both envelope pairs per clause:

```haskell
-- keiki-codec-json/src/Keiki/Codec/JSON/Event.hs (encodeClause)
  kindPair =
    [|
        $(keyE (kindFieldName opts))
            Aeson..= (T.pack $(stringE (wireKindOf opts (ecCtorName ec))) :: Text)
        |]
  versionPair =
    [|
      $(keyE (versionFieldName opts))
        Aeson..= ($(litE (IntegerL (fromIntegral (currentVersion opts)))) :: Int)
      |]
```

`kindOverrides` maps Haskell constructor names to deployed wire kinds. Unknown override keys,
duplicate resolved kinds, and payload collisions with the kind/version keys fail at splice time.
Renaming a Haskell constructor therefore does not require renaming its wire identity.

The decoder first reads the stored version (an absent `"v"` means version 1), rejects impossible or
future versions, migrates the whole envelope, and only then dispatches on the migrated wire kind.
That ordering lets an upcaster rewrite both payload keys and the discriminator.

```haskell
-- keiki-codec-json/src/Keiki/Codec/JSON/Event.hs (dispatch)
foldr
    ( \ec elseQ ->
        [|
            if $(varE kVar) == T.pack $(stringE (wireKindOf opts (ecCtorName ec)))
                then $(buildCtorDecode opts oVar ec)
                else $elseQ
            |]
    )
    [|Left ("unknown event kind: " <> T.unpack $(varE kVar) <> expectedKinds)|]
```

The discriminator key defaults to `"kind"`; the version key defaults to `"v"` and the current
version to 1.

### The no-silent-fallback contract

This is the load-bearing property. A payload field is encoded one of three ways, chosen by **field
name**:

```haskell
-- keiki-codec-json/src/Keiki/Codec/JSON/Event.hs
classify opts fn =
    case Map.lookup fn (fieldCodecOverrides opts) of
        Just fc -> Override fc
        Nothing
            | fn `Set.member` passthroughFields opts -> Passthrough
            | otherwise -> Unhandled
```

<Steps>
<Step>
**Override.** If the field name is a key of `fieldCodecOverrides`, the author-supplied `FieldCodec`
functions are spliced in. (In the fixture, `orderId` is overridden to encode as an `"ord-<n>"` string
rather than a bare integer.)
</Step>
<Step>
**Passthrough.** Else if the field name is in `passthroughFields`, the field's own `ToJSON`/`FromJSON`
instances are used directly.
</Step>
<Step>
**Unhandled.** Otherwise the field is *unhandled*, and `onMissingCodec` decides what happens — at
compile time.
</Step>
</Steps>

There is never a quiet generic guess. Adding a field to a payload record forces the author to make a
decision. The two policies:

```haskell
-- keiki-codec-json/src/Keiki/Codec/JSON/Event.hs
data OnMissingCodec
    = FailAtCompileTime
    | EmitTodoBindings
```

`FailAtCompileTime` (the default) aborts the splice with `fail`, listing every unhandled field
qualified by constructor and type:

```haskell
-- keiki-codec-json/src/Keiki/Codec/JSON/Event.hs
        FailAtCompileTime
            | not (null unhandled) ->
                fail $
                    "deriveEventCodecSkeleton: "
                        <> show tyName
                        <> " has field(s) with no provided codec and "
                        <> "onMissingCodec = FailAtCompileTime:\n"
                        <> intercalate
                            "\n"
                            [ "  - " <> nameBase c <> "." <> fn <> " :: " <> pprint ft
                            | (c, fn, ft) <- unhandled
                            ]
                        <> "\nAdd each field to fieldCodecOverrides or passthroughFields, "
                        <> "or set onMissingCodec = EmitTodoBindings."
```

`EmitTodoBindings` instead emits, for each unhandled field, a clearly-named `_todo_<Event>_<field>`
binding that *compiles* but whose body is an `error "TODO: …"`:

```haskell
-- keiki-codec-json/src/Keiki/Codec/JSON/Event.hs (mkTodoBinding)
    def <- funD nm [clause [] (normalB [|error $(stringE msg)|]) []]
```

So `EmitTodoBindings` lets the module build while leaving an obviously-named landmine: any actual
encode or decode of that field throws the named error rather than silently round-tripping through a
generic instance. Either way, the author is forced to confront the new field — that is the anti-drift
property in full.

### Additive defaults and structural migrations

`FieldCodec` has a third field, `fcOnMissing :: Maybe Name`. The `fieldCodec` smart constructor
creates a strict override; record-update `fcOnMissing` with a top-level constant when a newly added
field has a domain-safe historical default. A syntactic `Maybe t` passthrough has the built-in
missing-key default `Nothing`. All other fields remain strict.

Structural changes use `currentVersion` and `upcasters`. Each `(n, 'upgradeN)` entry names an
`Aeson.Value -> Either String Aeson.Value` function that rewrites one complete envelope from version
`n` to `n + 1`. `validateUpcasters` requires exactly one rung for every source version in
`[1 .. currentVersion - 1]` and rejects duplicate or out-of-range sources. Rung failures retain the
source version in their message.

This generated pipeline is intentionally one envelope to one envelope. Event splits and merges stay
at the application/event-store boundary. Event decoding ignores unknown keys for additive rolling
deployments; chapter 07's snapshot decoder rejects them because a snapshot must match one exact
shape.

<Callout type="warn">
This is what "sensitivity" means in keiki — **structural drift caught loudly, no silent fallback** — not
redaction. A payload that gains a field is a structural change, and the deriver refuses to paper over it.
</Callout>

The translator can only push a value into the solver if it knows how to *represent* that value in
SBV's symbolic universe. `Keiki.Symbolic` pins that knowledge in the `Sym` typeclass and a curated
runtime registry. This chapter reads both, from `keiki/src/Keiki/Symbolic.hs`.

## The `Sym` class

```haskell
-- keiki/src/Keiki/Symbolic.hs
class (SBV.SymVal (SymRep a), Typeable a) => Sym a where
    type SymRep a :: Type
    toSym :: a -> SymRep a
    fromSym :: SymRep a -> a
    symDefault :: a
```

<TypeTable
  type={{
    SymRep: { type: "Type (associated)", description: "The SBV-friendly carrier type the value is encoded as — e.g. Integer for every integer width, String for Text." },
    toSym: { type: "a -> SymRep a", description: "Push a concrete Haskell value into its SBV representation (used by symLit / extraction inputs)." },
    fromSym: { type: "SymRep a -> a", description: "Pull a concrete witness back out of a model value." },
    symDefault: { type: "a", description: "Fallback value symSatExt uses for a slot/field the model left unconstrained — sound because such slots satisfy the predicate at any value." },
  }}
/>

The `SBV.SymVal (SymRep a)` superclass is doing real work: it hands the translator `SBV.literal`,
`SBV.free`, and `SBV.unliteral` for the carrier type for free. The header spells out why
`symDefault` exists at all:

```haskell
-- 'symDefault' is consumed by 'symSatExt': when the solver's model
-- has no value for a slot or input field that the predicate did not
-- reference, the witness extractor falls back to 'symDefault'. Sound
-- because such slots are unconstrained — any value satisfies the
-- predicate.
```

We will see `symDefault` again in [chapter 05](/docs/keiki/walkthrough/symbolic-and-validation/05-symisbot-and-witness-extraction)
where `readModel` calls it on a model miss.

## The instances, and the unbounded-Integer over-approximation

`Bool` and `Integer` are identities — they encode as themselves:

```haskell
instance Sym Bool where
    type SymRep Bool = Bool
    toSym = id
    fromSym = id
    symDefault = False

instance Sym Integer where
    type SymRep Integer = Integer
    toSym = id
    fromSym = id
    symDefault = 0
```

Platform `Int` encodes as the unbounded mathematical `Integer` because SBV has no arbitrary-size
`SInt`. That means `Int` overflow is not modelled:

```haskell
{- | Encoded as 'Integer'. SBV does not provide an 'SInt'-of-arbitrary-
size; using 'Integer' means machine-width 'Int' wraparound is not modeled.
Guards whose truth depends on 'Int' overflow should use an explicit
fixed-width type instead.
-}
instance Sym Int where
    type SymRep Int = Integer
    toSym = fromIntegral
    fromSym = fromIntegral
    symDefault = 0
```

The six explicit fixed-width types take a different path. Their `SymRep` is the same fixed-width
type, so SBV applies the same modular wraparound as Haskell:

```haskell
-- The fixed-width instances use SBV's matching bit-vector representations, so
-- their arithmetic has exactly the same modular wraparound as Haskell.
```

For example, money modelled as `Word64` remains a 64-bit unsigned value in the solver:

```haskell
-- | Money and large counts, modeled as an exact 64-bit unsigned value.
instance Sym Word64 where
    type SymRep Word64 = Word64
    toSym = id
    fromSym = id
    symDefault = 0
```

<Callout type="warn">
Use an explicit fixed-width integer when overflow is part of the domain. Only platform `Int` is
over-approximated by `Integer`; `Word8`/`Word16`/`Word32`/`Word64` and `Int32`/`Int64` preserve
their concrete bit widths.
</Callout>

### Text and time

`Text` encodes through SBV's string theory. `UTCTime` uses integer picoseconds since the Unix epoch,
which preserves `NominalDiffTime`'s fixed-point precision:

```haskell
-- | 'Text' is encoded as Haskell 'String' for SBV's 'SString' theory.
instance Sym Text where
    type SymRep Text = String
    toSym = T.unpack
    fromSym = T.pack
    symDefault = T.empty

-- | 'UTCTime' is encoded as picoseconds since the Unix epoch. The time
-- library's 'NominalDiffTime' uses a fixed-point picosecond representation, so
-- this 'Integer' encoding is lossless while remaining well supported by z3.
instance Sym UTCTime where
    type SymRep UTCTime = Integer
    toSym t =
        let MkFixed picoseconds =
                nominalDiffTimeToSeconds (utcTimeToPOSIXSeconds t)
         in picoseconds
    fromSym picoseconds =
        posixSecondsToUTCTime
            (secondsToNominalDiffTime (MkFixed picoseconds))
    symDefault = posixSecondsToUTCTime 0
```

## The runtime registry: discovering evidence from `Typeable`

The translator works over a `Term ... r` whose `r` is only known to be `Typeable`. To translate an
equality it needs to *recover* the `Sym` instance for `r` at runtime. That is what the `SymDict`
reified dictionary and `discoverSym` do:

```haskell
data SymDict r where
    SymDict :: (Sym r) => SymDict r

discoverSym :: forall r. (Typeable r) => Maybe (SymDict r)
discoverSym
    | Just HRefl <- eqTypeRep (typeRep @r) (typeRep @Bool) = Just SymDict
    | Just HRefl <- eqTypeRep (typeRep @r) (typeRep @Int) = Just SymDict
    | Just HRefl <- eqTypeRep (typeRep @r) (typeRep @Integer) = Just SymDict
    | Just HRefl <- eqTypeRep (typeRep @r) (typeRep @Text) = Just SymDict
    | Just HRefl <- eqTypeRep (typeRep @r) (typeRep @UTCTime) = Just SymDict
    | Just HRefl <- eqTypeRep (typeRep @r) (typeRep @Word64) = Just SymDict
    -- ... Word32, Word16, Word8, Int64, Int32 ...
    | otherwise = Nothing
```

`eqTypeRep` compares the requested type's `TypeRep` against each curated type; a match returns the
`HRefl` proof that brings the `Sym r` instance into scope, packed as `SymDict`. A miss is `Nothing`,
and the translator's fallback (a fresh opaque variable) keeps the answer sound but imprecise. The
registry is closed and hand-maintained — exactly the curated set the
[why-smt explanation](/docs/keiki/explanation/why-smt) calls out.

`SymbolicSpec` checks the registry's membership directly, including the deliberate miss:

```haskell
-- keiki/test/Keiki/SymbolicSpec.hs
it "discovers Sym Word64"  $ symKnown (Proxy @Word64)  `shouldBe` True
-- ...
it "rejects unknown types" $ symKnown (Proxy @())      `shouldBe` False
```

### Two companion registries: ordering and arithmetic

Equality only needs `Sym`. A *comparison* (`PCmp`) additionally needs SBV ordering evidence, and a
*computed* operand (`TArith`) needs SBV numeric evidence. Those are reified by `SymOrdDict` /
`discoverSymOrd` and `SymNumDict` / `discoverSymNum`:

```haskell
data SymOrdDict r where
    SymOrdDict :: (Sym r, SBV.OrdSymbolic (SBV.SBV (SymRep r))) => SymOrdDict r

data SymNumDict r where
    SymNumDict :: (Sym r, Num (SBV.SBV (SymRep r))) => SymNumDict r
```

Both registries are narrower than `discoverSym`. `discoverSymOrd` covers the numeric types and
`UTCTime` (epoch picoseconds are orderable) but deliberately drops `Bool` and `Text` — ordering a boolean
guard is meaningless and `SString` ordering is out of scope. `discoverSymNum` covers only the numeric
types, dropping `Bool`, `Text`, and `UTCTime`. The source explains the philosophy of the miss case,
which is the same one-directional fallback as `discoverSym`:

```haskell
-- 'Nothing' makes the 'PCmp' translator fall back to a fresh opaque
-- 'SBool', exactly as 'goEq' does for non-'Sym' operands — sound, just
-- imprecise.
```

These three `discover*` functions are the only place a type's symbolic capabilities are decided.
Everything downstream — the translator in the next chapter, the analyses after it — routes through
them. If a type is absent, the worst that happens is a fresh opaque variable and a lost-precision
answer that still errs toward a false alarm rather than a missed bug.

These are **ordered source tours**: numbered chapters that read keiki's real Haskell source to teach
how each capability is built, end to end. They are written for a developer who wants to contribute to
keiki — or simply to gain real confidence in how it works — rather than just call its API. There is
one tour per capability; the chapters for each tour are authored by that capability's documentation
plan.

<Callout type="warn">
**The Decider façade is legacy.** It exists so callers arriving from Jérémie Chassaing's *Functional
Event Sourcing Decider* world see a familiar record shape. The *recommended* surface for keiki is the
transducer's own verbs — `step` / `stepEither` for the forward step, `delta` / `omega` for the
projected pieces, and `reconstitute` for replay. Prefer those. This chapter reads `toDecider` to
explain what the façade does and where it stops short; it does not promote it as the surface to build
on.
</Callout>

The Decider record is the small four-field shape Chassaing popularised:

```haskell
-- keiki/src/Keiki/Decider.hs (the shape callers know)
-- data Decider c e s = Decider
--   { decide       :: c -> s -> [e]
--   , evolve       :: s -> e -> s
--   , initialState :: s
--   , isTerminal   :: s -> Bool
--   }
```

`Keiki.Decider` projects a `SymTransducer` onto this shape. We read the shipped record, the field-by-
field correspondence, the *defensive* `evolve`, and the one semantic gap that survives the projection.

## The shipped record has five fields, not four

The record keiki actually ships adds a fifth field — `evolveStreaming` — and a fourth type parameter
for the streaming-state carrier:

```haskell
-- keiki/src/Keiki/Decider.hs
data Decider c e s s_streaming = Decider
  { decide          :: c -> s -> [e]
  , evolve          :: s -> e -> s
  , evolveStreaming :: s_streaming -> e -> Maybe s_streaming
  , initialState    :: s
  , isTerminal      :: s -> Bool
  }
```

<TypeTable
  type={{
    decide: { type: "c -> s -> [e]", description: "Forward step: events emitted for a command in a state. Built on omega, so it can return two or more events for one command." },
    evolve: { type: "s -> e -> s", description: "Letter-only replay: fold one event back into the state. Built on applyEvent; defensive (returns the input state on failure)." },
    evolveStreaming: { type: "s_streaming -> e -> Maybe s_streaming", description: "Streaming replay across multi-event edges, staying InFlight between events. Built on applyEventStreaming; the Maybe is explicit." },
    initialState: { type: "s", description: "The transducer's initial vertex paired with its initial register file." },
    isTerminal: { type: "s -> Bool", description: "Final-state predicate, consulting isFinal and ignoring the register file." },
  }}
/>

Why the extra field? Because a single keiki edge can emit *two or more* events (the EP-19 widening of
`output` to a list). `decide` returns that full list directly. But the plain `evolve` is *letter-only*
— it handles edges with output of length 0 or 1. To replay a length-2+ edge event-by-event you must
pass through an intermediate "I just observed event 1, expecting event 2 next" state, and that is what
`evolveStreaming` exposes via `InFlight`. The two evolve fields agree on length-0/1 commands and
diverge only on length-2+, where the streaming path stays `InFlight` between events.

## `toDecider`: the projection

```haskell
-- keiki/src/Keiki/Decider.hs
toDecider
  :: (BoolAlg phi (RegFile rs, ci), Eq co)
  => SymTransducer phi rs s ci co
  -> Decider ci co (s, RegFile rs) (InFlight s co, RegFile rs)
toDecider t = Decider
  { decide = \cmd (s, regs) -> omega t s regs cmd
  , evolve = \(s, regs) ev -> case applyEvent t s regs ev of
      Just (s', regs') -> (s', regs')
      Nothing          -> (s, regs)
  , evolveStreaming = \(w, regs) ev -> applyEventStreaming t w regs ev
  , initialState    = (initial t, initialRegs t)
  , isTerminal      = \(s, _regs) -> isFinal t s
  }
```

The state carriers are the keiki-shaped tuples, not bare vertices. The letter-replay carrier is
`(s, RegFile rs)` and the streaming carrier is `(InFlight s co, RegFile rs)` — keiki edge guards depend
on the register file as well as the control vertex, so the register file has to travel inside the
Decider's `s`. The field-by-field correspondence the source documents:

```haskell
-- keiki/src/Keiki/Decider.hs (from the haddock)
-- decide d cmd (s, regs)        -- = omega t s regs cmd            (EP-19 widened)
-- evolve d (s, regs) ev         -- = letter-only applyEvent;
--                               --   on length-2+ edges, falls back
--                               --   to the input state defensively
-- evolveStreaming d ws ev       -- = applyEventStreaming, returning
--                               --   the wrapped state mid-chain
-- initialState d                -- = (initial t, initialRegs t)
-- isTerminal   d (s, _regs)     -- = isFinal t s
```

## The defensive `evolve`

`applyEvent` returns `Nothing` when an event cannot be replayed letter-by-letter from `(s, regs)`. The
Chassaing signature for `evolve` is `s -> e -> s` — *not* `Maybe`. To keep that signature, `toDecider`
absorbs the failure by **returning the input state unchanged**:

```haskell
-- keiki/src/Keiki/Decider.hs (the defensive branch)
evolve = \(s, regs) ev -> case applyEvent t s regs ev of
    Just (s', regs') -> (s', regs')
    Nothing          -> (s, regs)
```

This is a deliberate trade. A caller who needs failure to be *visible* should use `evolveStreaming`
(whose `Maybe` is explicit) or `Keiki.Core.applyEvents` (which returns `Nothing` on the first replay
failure across a chunk). `evolve` is the convenient-but-lossy field.

`DeciderSpec` exercises the streaming field's mid-chain behaviour directly. Replaying the head event of
the collapsed two-event entrance lands in an `InFlight` state carrying the expected tail event:

```haskell
-- keiki/test/Keiki/DeciderSpec.hs
it "Settled PotentialCustomer ⊢ RegistrationStarted → InFlight RequiresConfirmation [ConfirmationEmailSent]" $ do
  let d           = toDecider userReg
      (_, regs0)  = initialState d
      ev          = RegistrationStarted
                      (RegistrationStartedData "dave@x" "U2V3" (t 0))
  case evolveStreaming d (Settled PotentialCustomer, regs0) ev of
    Just (InFlight RequiresConfirmation [ConfirmationEmailSent _], _) -> pure ()
    Just (other, _)              ->
      expectationFailure ("expected InFlight RequiresConfirmation [...], got " <> show other)
    Nothing -> expectationFailure "evolveStreaming returned Nothing"
```

## The ε-edge gap

One semantic gap survives the projection. An **ε-edge** is an edge whose `output` is `[]`: the
transducer changes state without emitting an event. From the Decider record's point of view, `decide`
returning `[]` is indistinguishable from "no event happened" — so the façade *cannot* drive an ε-edge.
`DeciderSpec` pins this exactly. Driving `userReg` to `RequiresConfirmation` and then issuing the silent
GDPR command yields no event from `decide`, and since `evolve` over an empty event list is a no-op, the
façade leaves the state put:

```haskell
-- keiki/test/Keiki/DeciderSpec.hs
it "ε-edge limitation: GDPR from RequiresConfirmation yields [] from decide" $ do
  let d         = toDecider userReg
      ...
      evs       = decide d gdprCmd preGdpr
      afterGdpr = foldl (evolve d) preGdpr evs
  ...
  evs                `shouldBe` []
  -- The ε-edge limitation: with no event, evolve is a no-op, so
  -- the façade leaves the state at RequiresConfirmation even
  -- though the keiki delta would transition to Deleted.
  fst afterGdpr      `shouldBe` RequiresConfirmation
```

The companion test makes the point that the limitation is at the *façade boundary*, not in the
transducer — `delta` drives the very same ε-edge to `Deleted`:

```haskell
-- keiki/test/Keiki/DeciderSpec.hs
it "ε-edge cross-check: delta does transition the same input to Deleted" $ do
  ...
  case delta userReg vAtRC regsAtRC gdprCmd of
    Just (vNext, _) -> vNext `shouldBe` Deleted
    Nothing         -> expectationFailure "delta returned Nothing"
```

<Callout type="info">
This is the operational reason to prefer the transducer surface. When ε-driven transitions matter, use
`Keiki.Core.delta` / `Keiki.Core.step` directly — they see the state change the Decider record's
event-shaped vocabulary cannot express.
</Callout>

The full export surface of the façade is catalogued in the [Decider reference](/docs/keiki/reference/decider).

The types and functions on this page live in `Keiki.Decider`. They project a keiki (継起)
`SymTransducer` onto the small record that Jérémie Chassaing's *Functional Event Sourcing Decider*
popularised, so code written against that shape can drive a keiki aggregate.

<Callout type="warn">
The `Decider` is a **legacy compatibility façade**, not keiki's recommended API. The recommended
surface is the transducer itself — `step` / `stepEither` / `delta` / `omega` / `reconstitute` in
[`Keiki.Core`](/docs/keiki/reference/core). The façade **silently no-ops on empty-output (ε)
edges**: an edge that emits no event advances the underlying machine, but the façade's `decide`
returns `[]` and `evolve []` leaves the state untouched. Reach for the façade only when you are
porting Chassaing-shaped code; reach for the transducer for everything else.
</Callout>

For the "why" behind the façade and a decision guide, see the conceptual companion
`/docs/keiki/explanation/decider-facade-and-when-to-use-it`.

## `Decider`

The shipped record has **five fields** and **two state parameters**: a letter-replay state `s` and a
streaming-replay state `s_streaming`.

<TypeTable
  type={{
    decide: { type: "c -> s -> [e]", description: "Forward step: emit the full event list a command produces in this state (built on omega)" },
    evolve: { type: "s -> e -> s", description: "Letter-only inverse step; defensive — returns the input state unchanged on replay failure" },
    evolveStreaming: { type: "s_streaming -> e -> Maybe s_streaming", description: "Streaming inverse step that threads the InFlight wrapper across multi-event edges" },
    initialState: { type: "s", description: "The start state" },
    isTerminal: { type: "s -> Bool", description: "Final-state predicate" },
  }}
/>

```haskell
data Decider c e s s_streaming = Decider
  { decide          :: c -> s -> [e]
  , evolve          :: s -> e -> s
  , evolveStreaming :: s_streaming -> e -> Maybe s_streaming
  , initialState    :: s
  , isTerminal      :: s -> Bool
  }
```

<Callout type="warn">
The module's own haddock and keiki's `docs/research/*` notes still quote a **stale four-field**
Chassaing record (`decide` / `evolve` / `initialState` / `isTerminal`). The **shipped** record is the
five-field shape above — `evolveStreaming` was added for streaming replay. Trust the source, not the
prose.
</Callout>

## `toDecider`

Project a `SymTransducer` onto the `Decider` record.

```haskell
toDecider
  :: (BoolAlg phi (RegFile rs, ci), Eq co)
  => SymTransducer phi rs s ci co
  -> Decider ci co (s, RegFile rs) (InFlight s co, RegFile rs)
```

The two concrete state carriers fall out of the transducer's shape:

- The **letter-replay** carrier is the pair `(s, RegFile rs)`. keiki edge guards read the register
  file as well as the control vertex, so replay has to carry both — the bare vertex is not enough to
  decide which edge fires. (See
  [Registers vs. state](/docs/keiki/explanation/registers-vs-state).)
- The **streaming-replay** carrier is `(InFlight s co, RegFile rs)`. Replaying event-by-event across
  a length-2-or-more edge intrinsically passes through a mid-chain "I just observed event 1,
  expecting event 2 next" wrapper, which is what `InFlight` captures.

### How each field is built

<Steps>

<Step>
**`decide (s, regs) cmd = omega t s regs cmd`** — returns the **full** event list directly. A single
command can yield two or more events end-to-end because `omega` reads the widened edge output
(`[OutTerm rs ci co]`) introduced by EP-19. A prior `toMultiDecider` / `DriverConfig`
state-refinement path was **retired** by EP-19's output-widening; multi-event behaviour is now
first-class in the AST and needs no scaffolding.
</Step>

<Step>
**`evolve (s, regs) ev`** is built on `applyEvent` (the letter-only inverse step) and is
**defensive**. To keep the non-`Maybe` Chassaing signature, it returns the input state unchanged when
replay fails:

```haskell
evolve = \(s, regs) ev -> case applyEvent t s regs ev of
  Just (s', regs') -> (s', regs')
  Nothing          -> (s, regs)
```

`applyEvent` handles edges with output length 0 or 1; on a length-2-or-more edge replayed letter at a
time it cannot settle, so `evolve` falls back to the input state. Callers that want strict replay use
`evolveStreaming` (whose `Maybe` is explicit) or `Keiki.Core.applyEvents` (which returns `Nothing` on
the first failure across a chunk).
</Step>

<Step>
**`evolveStreaming (w, regs) ev = applyEventStreaming t w regs ev`** threads `InFlight s co` across a
multi-event edge, returning the wrapped state mid-chain and the settled state once the last event of
the chain arrives.
</Step>

<Step>
**`initialState = (initial t, initialRegs t)`** and **`isTerminal (s, _) = isFinal t s`** — the
start state pairs the initial vertex with the initial register file; terminality ignores the
register file and consults `isFinal`.
</Step>

</Steps>

## The deliberate ε-edge gap

An **ε-edge** emits no output — `omega` returns `[]`. The underlying transducer still advances state
across it (via `delta`), but the façade cannot: `decide` returns `[]`, and `evolve` applied to an
empty list of events is a no-op, so the state is left untouched. From the `Decider` record's
perspective, an ε transition is indistinguishable from "no event happened".

<Callout type="warn">
Use the transducer directly — `delta` / `step` from
[`Keiki.Core`](/docs/keiki/reference/core) — when **ε transitions matter**, when you need **strict
`Maybe` replay**, or when you need **streaming**. The façade is appropriate only for Chassaing-shaped
code on aggregates with no silent edges.
</Callout>

## Tested behaviour

`test/Keiki/DeciderSpec.hs` anchors the contract against the User Registration fixture:

- **Canonical round-trip.** `foldl` of `decide`-then-`evolve` over the canonical command sequence
  lands in `(Deleted, expectedSnapshot)`, the same end-state `reconstitute` produces from the
  opposite direction; `isTerminal d` then reports `True`.
- **Multi-event first command.** After EP-19's M7 collapse, the entrance is a length-2 edge, so
  `decide d (head canonicalCmds) (initialState d)` returns both events in declaration order —
  `[RegistrationStarted _, ConfirmationEmailSent _]`.
- **The ε-edge limitation, and its cross-check.** Driving the aggregate to `RequiresConfirmation`
  and then issuing `FulfillGDPRRequest` (an ε-edge), `decide` returns `[]` and the façade leaves the
  state at `RequiresConfirmation`. The companion case confirms `delta userReg` on the same input
  *does* transition to `Deleted` — the limitation lives at the façade boundary, not in the
  transducer.
- **`evolveStreaming` `InFlight` cases.** Replaying `RegistrationStarted` from
  `Settled PotentialCustomer` yields `InFlight RequiresConfirmation [ConfirmationEmailSent _]` (the
  mid-chain wrapper carrying the expected tail event); replaying `AccountDeleted` from
  `Settled Confirmed` settles directly to `Settled Deleted`.

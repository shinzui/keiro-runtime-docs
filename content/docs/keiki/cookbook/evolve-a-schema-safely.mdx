---
title: "Evolve a schema safely"
description: "Reconcile old wire events with a changed schema using an explicit upcaster at the event-store boundary, and let the shape hash invalidate stale snapshots."
---

A cookbook recipe is a short, copy-paste solution to one problem. Here: a deployed schema
change must still read events written by the old code.

## Problem

The aggregate's event or register-file shape changed. Events already in the store were written
against the old shape; snapshots were taken against the old register file. The new transducer
must replay both without silently producing wrong state.

## Solution

Use the generated event codec's two compatibility mechanisms:

- For a purely additive field, keep the current envelope version and define its missing-key
  behavior: a syntactic `Maybe t` passthrough defaults to `Nothing`, or a `FieldCodec` can name a
  constant through `fcOnMissing`.
- For a structural rewrite, bump `currentVersion` and provide every one-envelope-to-one-envelope
  rung in `upcasters`.
- Preserve a deployed constructor's discriminator with `kindOverrides`; do not make a Haskell rename
  a wire rename.

```haskell
missingDiscount :: Discount
missingDiscount = Discount 0

discountCodec :: FieldCodec
discountCodec =
  (fieldCodec 'discountToJSON 'discountFromJSON)
    { fcOnMissing = Just 'missingDiscount }

upcastQuantityV1 :: Aeson.Value -> Either String Aeson.Value
upcastQuantityV1 value = case value of
  Aeson.Object object -> case KeyMap.lookup (Key.fromString "qty") object of
    Nothing -> Left "missing qty"
    Just quantityValue ->
      Right
        (Aeson.Object (KeyMap.insert (Key.fromString "quantity") quantityValue object))
  _ -> Left "expected an object"

additiveOptions =
  defaultEventCodecOptions
    { fieldCodecOverrides = Map.fromList [("discount", discountCodec)]
    , passthroughFields = Set.fromList ["orderId", "note"]
    }

structuralOptions =
  defaultEventCodecOptions
    { passthroughFields = Set.fromList ["quantity"]
    , kindOverrides = Map.fromList [("QuantityRecorded", "quantity.recorded")]
    , currentVersion = 2
    , upcasters = [(1, 'upcastQuantityV1)]
    }
```

The decoder treats a missing `"v"` as version 1, rejects a future version, runs applicable rungs in
ascending order, then dispatches on the migrated wire kind and applies field defaults. The splice
requires exact upcaster coverage of `[1 .. currentVersion - 1]`, so a historical version cannot
silently skip a migration.

<Callout type="warn">
  Keiki's generated upcasters map **one JSON envelope to one JSON envelope**. If one historical event
  must split into several current events (or several must merge), perform that transformation at the
  application/event-store boundary and preserve the resulting replay order explicitly.
</Callout>

## Protect replay and snapshots

Run the replay-safety validation against the **current** event shape. If a migration removes data an
edge still needs, the guard/input-read checks report it before production replay.

Treat snapshots differently from events. A snapshot is a disposable cache and must match both the
consumer codec version and `regFileShapeHash`, which covers only the ordered register slot names and
canonical types. A mismatch discards the snapshot and replays the event log from event 0. Keiki 0.2
deliberately changes every non-empty 0.1 shape hash once because built-in type names are now pinned;
let that benign cache miss happen and write a fresh snapshot after replay.

Finally, freeze representative bytes. `Keiki.Codec.JSON.GoldenFileSpec` demonstrates the pattern:
keep current event envelopes, version-absent additive history, every migration-era version, register
snapshots for the chosen encoder path, and the canonical shape/hash as checked-in fixtures. Regenerate
only for an intentional compatibility decision and review the byte diff.

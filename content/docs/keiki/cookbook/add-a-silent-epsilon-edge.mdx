---
title: Declare an intentionally silent edge
description: Use noEmit explicitly, and keep persisted state changes replayable.
---

Keiki 0.2 requires every builder edge to declare whether it emits. `B.noEmit` is the explicit choice
for an output-free edge, but output-free state changes are not reconstructible from an event log.

## Problem

A command should be accepted without changing durable state or producing an event, or a transducer is
used only in memory and deliberately has no replay contract.

## Solution

A replay-neutral self-loop can explicitly accept an idempotent command:

```haskell
B.from Completed do
  B.onCmd inCtorAcknowledgeAgain $ \_d -> B.do
    B.noEmit
    B.goto Completed
```

This edge keeps the same vertex, uses `UKeep` because there are no slot writes, and emits nothing.
Forward execution and event replay therefore agree: both leave the aggregate unchanged.

For an in-memory-only state machine, `noEmit` may also accompany a state change, but make that boundary
explicit in the owning module and do not persist only its emitted outputs as an event log.

For a persisted aggregate, emit a domain event whenever the vertex or registers change:

```haskell
B.from Active do
  B.onCmd inCtorDelete $ \d -> B.do
    B.slot @"deletedAt" .= d.at
    B.emit wireDeleted DeletedTermFields { at = d.at }
    B.goto Deleted
```

<Callout type="info">"jitsurei" (実例) is the family's word for a runnable worked example.</Callout>

## How it works

`B.noEmit` sets the builder's output-decision flag while leaving the output list empty. Without
`emit`, `emitWith`, or `noEmit`, `buildTransducerEither` returns a located
`DefectMissingOutputIntent`.

Then run the replay-safety umbrella:

```haskell
validateTransducer defaultValidationOptions aggregate
```

`StateChangingEpsilon` identifies an output-free edge that changes vertex, can write registers, or
does both. The check is conservative about writes: any `USet` counts even if a particular command
writes the value already present. A self-loop with `UKeep` is clean.

See [the builder reference](/docs/keiki/reference/builder) for `noEmit` and
[Validate](/docs/keiki/reference/validate) for the durable replay checks.

---
title: "Derive a JSON event codec"
description: "Derive a pinned, versioned JSON event envelope with additive defaults and whole-envelope migrations, then expose its wire kinds to keiro."
---

You have an event sum type and want a JSON codec whose `kind` discriminator and per-field handling
are explicit — no quiet generic guess for a field you forgot to think about. The
`deriveEventCodecSkeleton` splice in `keiki-codec-json` derives the codec and forces a compile-time
decision for every payload field.

<Callout type="info">
  Assumes your event type is a sum where each constructor wraps a single record payload (or is a
  no-arg singleton), as in the order example below.
</Callout>

## Goal

Derive a versioned event envelope, pin constructor wire kinds, choose field codecs and missing-key
defaults, and provide explicit migrations for historical envelopes.

## Derive the skeleton

`deriveEventCodecSkeleton opts ''EventType` emits five top-level bindings, prefixed by the type name
with its first letter lower-cased. For `data OrderEvent = Placed PlacedData | Shipped ShippedData |
Cancelled`, the splice emits `orderEventToJSON`, `orderEventFromJSON`, `orderEventEventTypes`, and
`orderEventKindMap`, plus `orderEventSchemaVersion`. Each constructor encodes to an object carrying
the resolved `"kind"`, an in-band `"v"`, and its payload fields:

```haskell
{-# LANGUAGE TemplateHaskell #-}

import Keiki.Codec.JSON.Event
  ( EventCodecOptions (..)
  , defaultEventCodecOptions
  , deriveEventCodecSkeleton
  , fieldCodec
  )

$( deriveEventCodecSkeleton
     defaultEventCodecOptions
       { fieldCodecOverrides =
           Map.fromList [("orderId", fieldCodec 'orderIdToJSON 'orderIdFromJSON)]
       , passthroughFields = Set.fromList ["qty", "trackingNo"]
       , kindOverrides = Map.fromList [("Placed", "order.placed")]
       }
     ''OrderEvent
 )
```

At the defaults, `currentVersion = 1` and `versionFieldName = "v"`. The `Placed` value now carries
`{"kind":"order.placed","v":1,...}`. Pin every deployed kind whose Haskell constructor may be
renamed; the splice rejects unknown override keys, duplicate wire kinds, and payload fields that
collide with the kind or version keys.

## Choose how each field is handled

A payload field is encoded one of three ways, chosen by **field name**:

<Cards>

<Card title="fieldCodecOverrides">
Keyed by field name. The author-supplied `FieldCodec` (an `fcEncode` name and an `fcDecode` name)
is spliced in. Use this when a field's JSON form differs from its `ToJSON` instance — e.g.
`orderId` rendered as an `"ord-<n>"` string.
</Card>

<Card title="passthroughFields">
A set of field names whose type already has aeson instances and may use them directly. Use this for
ordinary scalar fields like `qty` or `trackingNo`.
</Card>

<Card title="unhandled">
A field in neither set. There is no quiet fallback — `onMissingCodec` decides what happens (below).
</Card>

</Cards>

## The no-silent-fallback choice

Here "sensitivity" means **no silent fallback**: adding a field to a payload record forces the
author to make a decision at compile time rather than letting a generic instance quietly take over.
The `onMissingCodec` option governs an unhandled field:

```haskell
import Keiki.Codec.JSON.Event (OnMissingCodec (..))

-- default: abort the splice, listing every unhandled field
defaultEventCodecOptions { onMissingCodec = FailAtCompileTime }

-- opt-in: emit a named placeholder that compiles but throws if used
defaultEventCodecOptions { onMissingCodec = EmitTodoBindings }
```

`FailAtCompileTime` (the default) aborts the splice with a message naming each unhandled field,
e.g. `Placed.discount :: GHC.Types.Int`, and tells you to add it to `fieldCodecOverrides` or
`passthroughFields`. `EmitTodoBindings` instead emits a clearly-named `_todo_Placed_discount`
binding whose body is `error "TODO: provide a FieldCodec for Placed.discount :: Int"` — the module
compiles, but any actual encode/decode of that field throws the named error instead of guessing.

<Callout type="warn">
  Sensitivity here is structural-drift detection, not redaction. A payload field is never silently
  dropped or masked — it is either handled explicitly or it stops the build (or throws a named
  error). The negative-test procedure is documented in `Keiki.Codec.JSON.THEventSpec`.
</Callout>

## Default a new additive field

Keep `currentVersion` unchanged for a compatible field addition. A syntactic `Maybe t` listed in
`passthroughFields` decodes a missing key as `Nothing`. For a non-`Maybe` field, attach a top-level
constant through `fcOnMissing`:

```haskell
import Keiki.Codec.JSON.Event (FieldCodec (fcOnMissing), fieldCodec)

missingDiscount :: Discount
missingDiscount = Discount 0

discountCodec :: FieldCodec
discountCodec =
  (fieldCodec 'discountToJSON 'discountFromJSON)
    { fcOnMissing = Just 'missingDiscount }
```

Only an **absent** key selects the default. A present `null` or malformed value still runs the field
decoder. Ordinary non-`Maybe` passthrough fields and strict overrides remain required.

## Migrate a structural change

For a rename, type change, or other envelope rewrite, bump the schema version and provide the whole
`n -> n + 1` rung. This version-1 migration renames `qty` to `quantity`:

```haskell
upcastQuantityV1 :: Aeson.Value -> Either String Aeson.Value
upcastQuantityV1 value = case value of
  Aeson.Object object -> case KeyMap.lookup (Key.fromString "qty") object of
    Nothing -> Left "missing qty"
    Just quantityValue ->
      Right
        (Aeson.Object (KeyMap.insert (Key.fromString "quantity") quantityValue object))
  _ -> Left "expected an object"

$( deriveEventCodecSkeleton
     defaultEventCodecOptions
       { passthroughFields = Set.fromList ["quantity"]
       , currentVersion = 2
       , upcasters = [(1, 'upcastQuantityV1)]
       }
     ''QuantityEvent
 )
```

An absent version is read as version 1. The decoder rejects future versions, runs all applicable
rungs in order, and only then dispatches on the migrated kind. The splice requires exact coverage of
`[1 .. currentVersion - 1]`; a missing, duplicate, or out-of-range rung fails compilation. A rung
migrates one envelope to one envelope. Put an event split or merge at the application/event-store
boundary, before replay reaches keiki.

## Wire the surfaces into a keiro registry

`<event>EventTypes :: [Text]` lists resolved wire kinds in declaration order, and
`<event>KindMap :: [(Text, Text)]` pairs each constructor with its resolved kind. Both are plain
`Text` with no keiro import, so a keiro adapter module can consume them to build an `eventTypes`
registry without keiki depending on keiro:

```haskell
-- In a keiro-side adapter module. orderEventEventTypes is the emitted [Text].
orderRegistry :: [Text]
orderRegistry = orderEventEventTypes
  -- e.g. ["order.placed", "Shipped", "Cancelled"], the wire `kind` values keiro
  -- routes on. orderEventKindMap pairs each with its kind string for any
  -- ctor/kind divergence.
```

## Verify it worked

`Keiki.Codec.JSON.THEventSpec` pins the round-trip and the discriminator:
`orderEventFromJSON (orderEventToJSON placed) == Right placed`, and
`orderEventToJSON (Placed (PlacedData (OrderId 7) 3))` is
`{"kind":"order.placed","orderId":"ord-7","qty":3,"v":1}` — the pinned kind and `"ord-7"`
override both visible. `Keiki.Codec.JSON.GoldenFileSpec` freezes that document, a singleton envelope,
an additive pre-default document with no `"v"`, and both sides of a version-1-to-2 migration. These
goldens verify that old bytes still decode and current bytes do not drift accidentally.

Use these guides after completing [Build your first migration
plan](/docs/pg-migrate/tutorials/your-first-migration-plan). Each guide starts from a specific goal
and keeps library-owned migration definitions separate from application- and operator-owned policy.

<Cards>
  <Card title="Author an embedded manifest" href="/docs/pg-migrate/how-to/author-a-manifest" description="Create a strict manifest v1, write transactional or nontransactional SQL, enable recompilation tracking, and package the files." />
  <Card title="Export a library component" href="/docs/pg-migrate/how-to/export-a-library-component" description="Let a library own exact migration bytes and dependencies while leaving execution to the application." />
  <Card title="Compose multiple components" href="/docs/pg-migrate/how-to/compose-multiple-components" description="Build one explicit dependency-ordered application plan from library-owned components." />
  <Card title="Integrate the CLI" href="/docs/pg-migrate/how-to/integrate-the-cli" description="Mount all eight commands while retaining application ownership of credentials, streams, logging, and exits." />
  <Card title="Check migrations in CI" href="/docs/pg-migrate/how-to/check-migrations-in-ci" description="Layer compile-time membership, pure plan, manifest, fresh-database, upgrade, and PostgreSQL-matrix checks." />
  <Card title="Author a new migration" href="/docs/pg-migrate/how-to/author-a-new-migration" description="Create the next SQL file exclusively, append it atomically, and preserve applied history." />
  <Card title="Test with ephemeral PostgreSQL" href="/docs/pg-migrate/how-to/test-with-ephemeral-postgres" description="Apply the complete plan to a temporary server and assert behavior through a fresh Hasql connection." />
</Cards>

## Commands

| Command | Database | Important options |
| --- | --- | --- |
| `plan` | no | `--component`, `--migration`, `--json` |
| `list` | no | `--component`, `--migration`, `--json` |
| `check` | no | required `--manifest PATH`, `--json` |
| `status` | yes | display filters, `--database-url`, `--json` |
| `verify` | yes | display filters, `--database-url`, `--json` |
| `up` | yes | database, wait, statement-timeout, `--json` |
| `repair ID` | yes | `--mark-applied` or `--retry`, `--reason`, `--confirm`, execution options |
| `new` | no | `--manifest`, optional `--name`, required `--description`, `--json` |

Inspection filters affect rendered selection only. Complete validation determines the exit class;
`up` never exposes a subset target.

Execution overrides are explicit. Lock flags `--wait`, `--no-wait`, and
`--lock-timeout MILLISECONDS` are mutually exclusive. Timeout flags
`--statement-timeout MILLISECONDS` and `--no-statement-timeout` are mutually exclusive. Omitted flags
preserve application defaults. Positive durations are required.

## Exit classes

```haskell
data ExitClass
  = ExitSucceeded
  | ExitVerificationFailed
  | ExitUsageFailed
  | ExitExecutionFailed
```

The host application maps these to process exit codes.

## JSON schema v1

`jsonSchemaVersion == 1`. Every command object contains `schemaVersion`, `command`, and `ok`, plus
either `data` or structured `error`. Checksums are lowercase exact hex; timestamps are UTC; enum
spellings, nulls, and array order are stable.

Successful up, repair, and history-import data contains `cleanup_issues`. A non-empty array means the
primary operation succeeded durably but lock release or statement-timeout restoration needs
investigation. It must not be reclassified as command failure.

Rendered text and `Show` are diagnostic, not automation protocols.

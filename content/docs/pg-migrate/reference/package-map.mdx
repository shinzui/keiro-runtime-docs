---
title: "Package and module map"
description: "The six pg-migrate packages, stable facade modules, and their dependency roles."
---

| Package | Stable facade | Role |
| --- | --- | --- |
| `pg-migrate` | `Database.PostgreSQL.Migrate` | identifiers, migrations, components, plans, ledger, inspection, runner, repair, generic history import |
| `pg-migrate-embed` | `Database.PostgreSQL.Migrate.Embed` | manifest v1 validation, exact-byte TH embedding, exclusive authoring |
| `pg-migrate-cli` | `Database.PostgreSQL.Migrate.CLI` | reusable parser, dispatcher, text/JSON rendering, exit classes |
| `pg-migrate-import-codd` | `Database.PostgreSQL.Migrate.History.Codd` | read exact Codd V1–V5 ledger shapes and import evidence |
| `pg-migrate-import-hasql-migration` | `Database.PostgreSQL.Migrate.History.HasqlMigration` | read qualified `schema_migrations`, verify legacy MD5, import evidence |
| `pg-migrate-test-support` | `Database.PostgreSQL.Migrate.Test` | apply a plan to ephemeral PostgreSQL before a fresh-connection callback |

A migration-owning library normally depends on core plus embed. The final administrative executable
adds CLI. Import adapters belong only in predecessor-cutover tooling, and test support belongs only in
test dependencies.

`Database.PostgreSQL.Migrate.Embed.RecompilePlugin` is a stable GHC 9.12 build-time module loaded by
an `OPTIONS_GHC -fplugin=…` pragma; application code does not call its `plugin` binding.

Modules containing `Internal`, rendered `Show` text, fixtures, and implementation modules not listed
as facades are outside the semantic compatibility promise.
